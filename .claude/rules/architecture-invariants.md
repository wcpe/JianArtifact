# 架构不变量（防架构漂移）

> 以下是本项目锁定的架构约束（依据 `docs/ARCHITECTURE.md` 与 `docs/adr/`）。**违反任一条即为架构漂移。**
> 确需改变某条 → 先写新 ADR 取代旧决策、经确认后再改；**禁止在代码里静默违背**。

## 1. 分层与依赖方向

- 依赖方向单向、无环：`api` → (`auth` / `authz` / `repo` / `format`) → (`proxy` / `storage` / `meta`) → `config`；`format` 可依赖 `storage` / `meta` / `proxy`。
- 上层不得被下层反向依赖；下层细节不得泄漏到上层。
- `api` 层（axum 路由与中间件）保持薄：只做协议适配、认证/鉴权编排与错误转换，不写业务逻辑。
- `meta` 是元数据的唯一访问入口，其他模块不得绕过它直接读写 SQLite。

## 2. 简单优先（禁用外部重型件）

- 禁止引入外部数据库（PostgreSQL / MySQL / Redis 等）、消息队列、独立搜索引擎、容器编排等重型依赖。
- 元数据只用嵌入式 SQLite；缓存只用进程内结构；不引入外部缓存/协调组件。
- 制品 blob 存储 P1 仅本地文件系统（S3 兼容后端为 P2，先走 ADR 再落地）。
- 不在二进制内实现 L3/L4 体积型 DDoS 缓解：只做应用层（L7）防护，体积型攻击交前置反向代理 / CDN / WAF（见 ADR-0008）。
- 漏洞库采用本地离线镜像 + 坐标级本地匹配（见 ADR-0012）：不以"逐包外发到外部漏洞服务"作为默认路径，守"数据不外发"。
- 确需打破上述限制 → 先写新 ADR 经确认，禁止静默引入。

## 3. 真源与一致性

- SQLite 是元数据的唯一真源；blob 本体在文件系统（P2 对象存储），DB 仅存索引与 `sha256`。
- 制品写入顺序固定：先把 blob 落盘并校验 `sha256` 通过，再写元数据索引；失败回滚，不留孤儿索引。
- 同一事实只有一个权威来源，禁止把元数据复制成互为权威的多份。
- 密钥/凭据的真源是配置文件与环境变量，**绝不入库、绝不写入 DB 明文**；DB 只存引用（如 `upstream_auth_ref`）。
- 使用分析/统计数据是本机内部数据：默认不主动外发、不向外部遥测平台 phone-home；任何外部导出须运维显式开启、默认关闭（见 ADR-0009）。

## 4. 技术栈锁定

- 后端：Rust + `axum` + `tokio`；`#![forbid(unsafe_code)]`。
- 前端：React + Vite + TypeScript + Mantine（UI 组件库），经 `rust-embed` 在编译期嵌入单一二进制。
- 元数据库：嵌入式 SQLite（`sqlx`，bundled）。
- 运行形态：单一可执行二进制 + TOML 配置 + 环境变量覆盖；零外部运行时依赖。
- 换语言 / 换框架 / 换元数据存储 = 架构决策 → 走新 ADR，不擅自更换。

## 红线（出现即停止并先确认）

引入被禁重型件（外部 DB / MQ / Redis 等）· 反向跨层依赖或绕过 `meta` 直接读写 DB · 私有仓库对匿名 / 未授权放行 · 开放公开自助注册或任意人自助创建管理员 · 明文存储密码 / Token / 密钥 · 凭据入库或写入日志 · 主动外发遥测 / 使用数据 / 制品坐标到外部 · 在二进制内实现 L3/L4 体积型 DDoS 缓解 · 擅自更换技术栈 · 静默违背任一已接受 ADR。
