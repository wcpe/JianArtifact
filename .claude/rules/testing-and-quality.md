# 验证门与质量底线（防质量漂移）

> 项目特定的测试与质量要求。通用反模式禁令（上帝类、长方法、复制粘贴、魔法值、吞异常、资源泄露、循环依赖、静态可变单例等）仍然适用。

## 1. 验证门（权威判据）
- **每个变更必须过验证门才算完成**，判据以**入库真源**为准：① `docs/PRD.md` 验收标准对应项满足；② 本文 §2 高风险区相关测试通过；③ 受影响组件全部测试绿（各组件按本项目实际的测试 / 构建命令执行）。
- **完成以证据为准、不轻信报告**：声称"做完 / 修好 / 通过"前，必须出示上述各项的客观证据（命令 + 关键输出、失败用例从红转绿、有实机 / 集成 / 真实环境维度的附该维度复验结果）。拿不齐 = 未完成、继续做——不轻信任何"完成了"的口头报告（含子代理的汇报）。
- **被推翻就回流继续修**：声称完成后被实测 / 实机 / 用户验收推翻，即视为该变更未完成，**回到本任务对应技能的验证步骤继续修**，不另开新流程、不重标状态。
- **提交受验证门门控**：验证门未全绿不得 `git commit`（门控见 `git-commit.md` §3）。
- `.tmp/` 下实施计划的里程碑勾选**仅作开发期辅助**，不入库、不作权威判据（clone 后可能不存在）。
- **禁止**以注释、跳过、删除失败测试的方式让测试"通过"。
- 改功能代码前先跑相关测试确保通过；新增 / 改业务逻辑同步加测试。

### 1.1 测试分层（怎么分、在哪跑）
- **单元**：纯逻辑（核心算法 / 纯函数）—— 不连外部依赖，最快最多。
- **集成**：被测组件 + 真实外部依赖（测试库 / 容器）跑关键流程。
- **E2E**：按本项目编排（如 compose）拉起完整依赖，跑关键时序（见 ARCHITECTURE 时序与 PRD 验收）。
- **何时跑**：单元 / 集成随每次改动与 CI；E2E 在发版前至少跑一遍。

## 2. 必测的高风险区

下列六块是本项目最易出错、出错代价最高的部分，每块必须穷举其边界后方可放行。

### 2.1 鉴权判定矩阵
- 必须穷举 `public/private` 可见性 × 角色（Admin / User / 匿名） × ACL（读 / 写 / 无）的**全组合**判定结果，逐格断言放行或拒绝。
- 重点不漏项：**私有仓库对匿名一律拒绝**（返回 401/404，且不泄露资源存在性）；**写操作写权限边界**——仅命中写 ACL（或具备相应角色）方可写，读权限不得越权写。
- 认证中间件三种身份解析（Bearer / Basic / 会话）与匿名分支，分别走一遍鉴权矩阵，避免某一身份通道绕过判定。
- 检索鉴权过滤：跨仓库搜索与制品详情结果必须按读权限过滤——穷举匿名 / 无权用户检索私有仓库制品时，结果、计数与错误均不泄露其存在；越权详情访问返回 404。

### 2.2 各格式协议正确性
- Maven / npm / Docker（OCI registry v2） / Raw 四种格式的**元数据生成与校验和**必须符合各自原生协议规范。
- 必须与官方客户端**真机互通**：`mvn deploy` / `mvn get`、`npm publish` / `npm install`、`docker push` / `docker pull`、`curl PUT` / `curl GET` 端到端成功，且校验和（含 sha256）一致。
- hosted 与 proxy 两种仓库类型对每种格式分别验证，路径解析须按仓库名正确定位。
- 覆盖/不可变与删除语义：穷举各格式重复上传（Maven release 拒绝 409 / snapshot 覆盖、npm 已发布拒绝、Docker tag 覆盖、Raw 覆盖）与制品删除（hosted 删本体+索引、proxy 删缓存后可重新拉取）的行为正确性。
- 多校验和正确性：sha256 / sha1 / md5 / sha512 计算正确，与各格式 sidecar 内容及客户端独立计算一致；流式写入时边算边校验、不二次读盘。

### 2.3 代理缓存
- cache-miss 路径：从上游拉取 → 校验 → 落盘 → 写索引，全链成功且缓存命中后不再回源。
- **并发单飞合并**：同一制品的并发 cache-miss 请求必须合并为一次上游拉取，穷举竞态时序，断言不重复下载、不写坏缓存。
- 上游失败回退：上游不可用 / 超时 / 返回错误时按既定策略回退，不得把损坏内容写入缓存。
- 缓存一致性：落盘内容与索引记录（sha256、size）一致，不出现索引指向不存在或不匹配的 blob。

### 2.4 流式大文件
- 上传 / 下载走流式处理，断言**大文件不整体载入内存**（监控峰值内存，不随文件体积线性膨胀）。
- 传输中断 / 截断处理：连接中断、字节数不足、写入未完成时必须报错并清理，不留半截 blob。
- sha256 校验：落盘后校验 sha256，校验失败即拒绝并回滚，不写元数据。
- 上传大小上限：超过配置上限（`limits.max_artifact_size`）的上传被拒绝并返回 413，不写入半截 blob。

### 2.5 元数据事务
- 制品写入次序：**blob 先落盘并校验通过，再写元数据索引**；穷举落盘成功而写索引失败、写索引成功而落盘失败等分支，断言失败回滚、不留**孤儿索引**与孤儿 blob。
- 并发写一致性：同一仓库 / 同一 path 的并发写，穷举竞态，断言元数据最终一致、无重复或丢失记录。

### 2.6 凭据与会话
- Token 生命周期：过期、吊销（`revoked`）后的 Token 必须被拒绝；签发 / 列表 / 吊销链路正确，Token 仅以哈希存储、不回显明文。
- Basic / Bearer 边界：两种凭据通道的解析、错误格式、缺失凭据等边界分别覆盖，避免任一通道绕过认证。
- 密钥脱敏：密码、Token、上游凭据等敏感项不得进入日志、错误响应或 DB 明文，断言输出已脱敏。
- 会话生命周期：会话 / JWT 的 TTL 到期后被拒绝、刷新端点正确续期、登出 / 吊销即时失效；会话与 API Token 相互独立、不串味。
- 登录暴力破解防护：连续失败达阈值后账户 / IP 在时间窗内被锁定 / 限流，锁定到期自动恢复；穷举正常用户不被误锁、伪造来源（XFF）不绕过。
- 首启引导：仅当库中无任何用户时触发；穷举“已有用户时不得再次引导”“env 提供与随机口令两条路径”“引导后不开放公开自助注册”。

### 2.7 七层防护中间件（P2，实现时必测）
- 误杀（false positive）：限流 / 封禁 / CC 挑战 / WAF 规则不得误伤合法客户端，穷举正常高频访问、合法批量拉取等场景断言放行。
- 绕过（bypass）：穷举伪造来源 IP（XFF 头）、轮换 Token / 账号、慢速攻击、超大请求体等绕过手法，断言被识别与拦截。
- 热路径开销：防护中间件位于请求热路径，断言其开销可控、不显著拖慢正常请求。
- 边界一致性：封禁状态与限流计数在并发下一致；配置变更（阈值 / 规则 / 名单）即时生效且不误判。

### 2.8 使用分析采集（P2，实现时必测）
- 异步不阻塞：采集走异步通道，断言不阻塞、不拖慢主请求路径，采集失败不影响业务。
- 计数准确性：并发访问 / 下载下聚合计数准确、无丢失或重复。
- 数据治理与隐私：明细数据量级可控（不撑爆 SQLite）；断言统计数据不外发、不向外部 phone-home，外部导出默认关闭。

## 3. 质量底线
通用反模式底线（任何项目都适用）：
- 分层依赖单向、不反向穿透（防上帝类 / 防泄漏底层细节到上层）。
- 锁 / 临界区职责单一、不嵌套，高开销 IO 一律在锁外。
- 核心计算尽量做成无副作用纯函数，便于穷举测试。
- 多处写操作必须在事务内原子完成，**事务提交成功后**才触发对外副作用（通知 / 唤醒 / 发事件）。
- 中文分级日志（ERROR/WARN/INFO/DEBUG），不留 `println` / `console.log` 等临时调试输出。
- 不硬编码凭据 / 端口 / 超时等，走配置或常量。

本项目特定质量条款（结合架构不变量与领域特性）：
- **分层与依赖方向锁定**：`api` → (`auth`/`authz`/`repo`/`format`) → (`proxy`/`storage`/`meta`) → `config`，单向无环；`format` 依赖 `storage`/`meta`/`proxy`；严禁反向跨层依赖与环。
- **axum handler 保持薄**：HTTP 层只做请求解析、调用与响应封装，业务逻辑下沉到对应 service / 领域模块，不在 handler 里写业务。
- **格式处理器单一职责**：maven / npm / docker / raw 各处理器只负责自身格式协议，经统一 trait 抽象注册；不得把跨格式逻辑塞进单个处理器，也不得用 `if-else` / `switch` 按格式名分支堆叠可变逻辑。
- **锁外做 IO**：拉取上游、读写 blob、查 SQLite 等高开销 IO 一律置于临界区之外；临界区只保护内存态与单飞合并的归属判定，不在持锁期间阻塞 IO。
- **blob 先落盘再写索引**：制品写入必须先把 blob 落到存储并校验 sha256 通过，再写 `meta` 元数据索引；任何失败路径都要回滚，不留孤儿索引；SQLite 是元数据唯一真源，blob 本体在文件系统 / 对象存储，DB 仅存索引 + sha256。
- **安全脱敏**：密码用 argon2、Token 哈希存储；上游凭据等密钥的真源在配置 / env，DB 仅存引用（`upstream_auth_ref`），凭据 / 密钥绝不入库、不进日志、不进 DB 明文。
- **中文分级日志**：日志、调试输出一律中文，按 ERROR / WARN / INFO / DEBUG 分级；包含必要上下文（如请求 ID、仓库名），不打印凭据与隐私。
