# 架构设计：JianArtifact

> 系统当前真貌（HOW）。始终原地更新到现状；结构 / 机制变了就改它。

## 1. 定位与边界

JianArtifact 是一个用 Rust 编写、打包为单一可执行二进制的轻量级多格式制品库管理器。它原生支持多种主流包格式，内置多用户认证、全局角色与每仓库 ACL，支持公开/私有仓库隔离与匿名访客，零外部运行时依赖。

是什么：

- 一个面向制品分发的服务端，对外暴露两类接口——供 Web 控制台与脚本使用的管理 API，以及供包管理器客户端（mvn / npm / docker / curl 等）按各自原生协议访问的格式 API。
- 一个把前端（React + Vite + TypeScript，UI 组件库 Mantine）在编译期嵌入二进制的自包含程序，启动后即可提供 Web 控制台与所有端点，无需独立的静态资源服务器。
- 元数据由嵌入式 SQLite 持有，制品本体（blob）落在本地文件系统。

不是什么：

- 不是依赖外部数据库或中间件的系统：不需要独立的 PostgreSQL/MySQL/Redis、消息队列、独立搜索引擎或容器编排。
- 不是反向代理或 TLS 终结器：TLS 终结由可选的外部反向代理承担，本程序不内建。
- 不是 P2/P3 范围内能力的承载者：OIDC/LDAP、审计日志、指标端点、速率限制、迁移工具、group/virtual 聚合仓库、GC/保留策略、备份恢复，以及七层防护增强（并发/连接控制、慢速攻击防护、异常检测与自动封禁、CC 挑战、WAF 规则引擎、监控告警）、使用分析数据面板、权限增强（用户组/团队、细粒度权限动作）等均不在当前形态内。（S3 兼容对象存储后端为已实现的可选 opt-in blob 后端，默认仍走本地文件系统，见 ADR-0014。）
- 不是 L3/L4 体积型 DDoS 缓解设施：本程序只做应用层（L7）防护，体积型攻击交由前置反向代理 / CDN / WAF 承担。
- 不是对外遥测的上报方：使用分析数据落本地、不主动外发，不向外部遥测平台 phone-home。

外部边界：

- 上游制品仓库：proxy 类型仓库在缓存未命中时向上游拉取制品。
- 包管理器客户端：通过 Bearer Token 或 Basic Auth 访问格式端点。
- 可选反向代理：置于本程序之前做 TLS 终结。

## 2. 模块与依赖

系统由以下模块构成，各自职责单一，依赖方向单向且无环：

- `api`：axum 路由与中间件（认证、鉴权、请求 ID、统一错误处理）。HTTP 层保持轻薄，不写业务逻辑。
- `auth`：认证。处理本地用户名/密码、Bearer Token、Basic Auth；提供认证 provider 抽象（统一 `AuthProvider` trait，本地口令为默认且始终启用，OIDC 经授权码流接入，LDAP 经口令型 bind 校验接入）。OIDC（FR-34 / ADR-0016）经标准授权码流 + PKCE 接入；LDAP（FR-35 / ADR-0016）经口令型 `authenticate_password` 做 bind 校验接入，仅参与 Web 表单 / Basic Auth 口令登录。两者均在登录边界完成「外部身份 → 本地用户」映射后照常签发既有会话/JWT，下游四通道与鉴权矩阵不变。
- `authz`：授权。负责全局角色 + 每仓库可见性（public/private）+ 每仓库 ACL 的综合判定；ACL 动作细化为四级 `read` / `write` / `delete` / `admin`，高动作蕴含低动作（admin ⊇ delete ⊇ write ⊇ read）。用户对某仓库的有效权限为其直接 ACL ∪ 其所属各组的组 ACL（取并集后按动作蕴含判定，FR-49 / ADR-0007）；既有直接-ACL 判定结论不变。
- `repo`：仓库模型与生命周期（hosted/proxy 配置、可见性）。
- `format`：各格式处理器（maven/npm/docker/go/raw/pypi），经统一 trait 抽象注册。
- `proxy`：上游代理与缓存（拉取、落盘、单飞合并、上游失败回退）。
- `storage`：blob 存储抽象（`BlobStore` trait；默认本地文件系统 `LocalFsStore`，可选 opt-in 的 S3 兼容对象存储 `S3Store`，经 `BlobBackend` 枚举运行期分发，见 ADR-0014）。
- `meta`：SQLite 元数据访问层（users / repositories / repo_acl / tokens / artifacts 索引）。
- `web`：React + Mantine 前端 + `rust-embed` 静态资源嵌入与服务。控制台外壳（`AppLayout`）为 Mantine `AppShell`：**可折叠图标导航条**（默认窄、仅图标 + Tooltip/`aria-label`，可展开为图标+文字），管理类入口据 `useAuth().isAdmin` 角色门控显隐，导航 active 按路径段精确匹配（不串台）；信息密度基线 token 集中于 `theme/density.ts`（导航宽度 / padding / 卡片瘦身 / 栅格 gap），shell 与仪表盘页先行落地、其余页面的密度细化随后续 UX 重构 FR 跟进（FR-92 起的第四期 epic）。仓库浏览（FR-93）为 Nexus-like **左侧文件树 + 右侧制品详情面板**（`RepositoryDetailPage` 的「浏览」Tab）：树由 `listArtifacts` 索引经 `lib/browse.ts` 客户端逐级折叠、按格式显示专属 icon（`lib/formatIcon.tsx`），右侧 `ArtifactDetailPanel`（独立深链页 `/artifact` 复用同一组件）展示元数据 / 四校验和 / 后端使用方式片段，并由纯前端模板（`lib/coordinates.ts`，GAV 反解规则与后端 `Gav::from_path` 对齐）对 Maven 主构件生成多格式依赖坐标（Maven / Gradle / SBT / Ivy / Grape / Leiningen / PURL），另附 HTML View 外链（指向 FR-75 目录索引）与下载链接；坐标 / 链接生成纯前端、不改后端。跨仓库制品搜索（FR-94）入口前移到页眉全局搜索框（`AppLayout`）：输入关键字回车或防抖后跳转 `/search?q=`，`SearchPage` 经 URL `q` 自动调既有 `GET /search` 端点（结果仍按读权限后端过滤），命中经 `lib/searchTree.ts` 客户端折叠为「按仓库分组」的可展开树、每个仓库 / 命中按格式复用 `lib/formatIcon.tsx` 显示专属 icon、点击叶子进 `/artifact` 详情；纯前端、不改后端搜索语义。监控页（`/monitor`，仅 Admin，FR-99）为**跨域运行总览**：顶部分类切换（全部 / 主机 / 使用分析 / 防护 / 缓存 / 存储仓库）+ 时间范围切换（1h / 24h / 7d），下接 **KPI 指标行**（各域指标当前值）与**多指标时序网格**（每指标一张卡：标题 + 手搓零依赖 SVG 折线 `components/charts/LineChart` + 当前值，悬停看某时间节点取值），消费 FR-105 `GET /api/v1/monitor/metrics` 取时序点；指标元数据（10 指标键 / 分类 / 值格式 / 时间范围）集中于 `lib/metrics.ts` 单一真源，无数据源指标（如缓存命中率）优雅显示「暂无数据」。使用分析（FR-58，`/analytics`）/ 审计（FR-77，`/audit`）/ 防护监控（FR-78，`/protection-monitor`）为各自独立路由 + 导航入口（仅 Admin、复用既有页组件、数据层不改），不并入监控页。角色感知 UI（FR-95）把后端既有权限模型（FR-08 匿名读 public、ADR-0004 按 ACL）在前端呈现为**三层路由守卫**——公开层（`/`、`/repositories`、`/repository`、`/search`、`/artifact`，匿名可达、只读公开内容，`AppLayout` 不再整体包 `RequireAuth`）、需登录层（`/tokens`、`/upload`，`RequireAuth`）、需管理员层（其余管理路由，`RequireAuth` + `RequireAdmin`）；`AppLayout` 据登录态渲染匿名 shell（页眉「登录」按钮 + 仅公开导航项）或登录 shell（用户名 / 登出 + 按 `isAdmin` 门控），匿名落地 `/` 重定向至 `/repositories`；纯前端呈现、权限仍由后端强制，不做公开自助注册。控制台版本展示（FR-101）落在 `AppLayout` 外壳：导航项**下方**以 footer 小字块呈现当前版本号（取自公开 `GET /health` 的 `version`，经 api 层 `getHealth` 封装根路径 `/health`；所有用户含匿名可见，小灰字）+「开源许可」入口（带 icon、进 `/licenses`），**仅展开态显示、窄（收缩）态整块隐藏**；页眉 Logo 旁仅 Admin、且挂载时调一次 `GET /update/check` 返回 200 且 `update_available` 才显「更新: cur → latest」徽标（点击进 `/settings`，未启用 409 / 无更新 / 失败均静默不显），两次查询走后台、不阻塞外壳渲染、不轮询；纯前端、不改后端契约。**国际化 i18n（FR-111 / ADR-0034）**：前端经 `i18next` + `react-i18next`（`src/i18n` 初始化全局单例、`main.tsx` 与测试 `setup.ts` 装载），全站用户可见文案抽到 `src/i18n/locales/zh-CN/<ns>.ts`——**按命名空间分文件**（每页一个 ns + `common`/`nav`/`errors`/`auditActions`），组件 `useTranslation('<ns>')` 取 `t`、跨 ns 用 `t('common:x')`；**本期只交付 zh-CN**（默认且唯一、无切换 UI），结构为多语言预留。审计 / 近期活动动作 key（含点，如 `artifact.upload`）经 `tAuditAction` 归一（`.`→`_`）查 `auditActions` ns 译中文、未知回落原串。纯前端、不改后端 / API 契约。
- `config`：TOML 配置加载 + env 覆盖。
- `migrate`：P2 模块——Nexus OSS 迁移（在线 REST + 离线 blob store 双入口）。当前已实现**在线 REST API 入口**（FR-36）的发现 / 预览：连接在线 Nexus、经其 `service/rest/v1/repositories` 枚举可迁移仓库列表与基本元数据（名 / 格式 / 类型 / 上游地址）；REST 交互经 `NexusClient` trait 抽象、生产实现复用 reqwest 纯 rustls，访问凭据真源 env（`JIANARTIFACT_MIGRATE_<NAME>_USERNAME/PASSWORD`，DB 仅存引用、不入库不进日志）。同时已实现**离线 blob store 入口**（FR-37）的发现 / 预览：当源 Nexus 已下线、只剩其文件型 blob store 目录时，从给定本地目录解析磁盘布局（`content/` 分片目录 + 每个 blob 一份 `.properties` 元数据），按所属仓库枚举可迁移 blob 及基本元数据（坐标 / sha1 / 大小）；纯文件系统读取、解析逻辑为无副作用纯函数，软删 / 损坏 / 缺字段的元数据容错跳过。在此之上已实现 **proxy 仓库配置 + 缓存制品搬运**（FR-38）：据在线 REST 枚举的 proxy 仓库配置（仅 `type==proxy` 者，含映射格式与上游 `remoteUrl`）在本系统建仓（同名仓库复用、未实现格式或缺上游则整体跳过），再从离线 blob store 按仓库名取该仓库的缓存制品本体（成对的 `.properties` + `.bytes`，缺本体者跳过），经既有 `format::ArtifactService::ingest_cached` 流式写入缓存——复用通用制品机理的核心不变量「blob 先落盘并校验 sha256 再写元数据索引（`cached=true`），写索引失败回滚不留孤儿」；搬运幂等可重入（同坐标同内容跳过），单制品失败（路径非法 / 读本体失败 / 写入失败）记 WARN 后跳过、不中断整批，无须持久化迁移任务表。在此之上已实现 **hosted 仓库配置 + 完整制品搬运**（FR-39）：据在线 REST 枚举的 hosted 仓库配置（仅 `type==hosted` 者，含映射格式；同名仓库复用、未实现格式整体跳过）在本系统建 hosted 仓库（无上游地址），再从离线 blob store 按仓库名取该仓库的全部制品本体，经既有 `format::ArtifactService::ingest_hosted` 流式写入——同样守「blob 先落盘并校验 sha256 再写元数据索引」，区别在于落为正常 hosted 制品（`cached=false`）而非缓存，并据各格式覆盖 / 不可变策略处理重复搬运（同坐标不同内容且不可覆盖如 Maven release 则跳过该制品、不中断整批；可覆盖则落定新内容），超 `limits.max_artifact_size` 的制品按跳过处理（不留半截 blob）；搬运幂等可重入（同坐标同内容跳过），单制品失败记 WARN 后跳过、不中断整批。proxy 与 hosted 搬运共用格式映射与 blob 归一化逻辑（`migrate` 模块内 `pub(crate)` 复用），离线 blob 枚举与按仓库归组的编排同款。至此 Nexus 迁移框架（FR-36/37/38/39）完整。在此之上补齐 ADR-0006 在线入口的「搬运」（FR-82）：**Maven hosted 在线拉取迁移**——源 Nexus 在线时经其 `service/rest/v1/components`（`continuationToken` 分页）枚举所选 hosted 仓库的全部 asset，按各 asset 的 `downloadUrl` HTTP 流式下载、经 `format::ArtifactService::ingest_hosted` 落为本系统 hosted 制品（守同一「blob 先落盘并校验 sha256 再写元数据索引」不变量），落定后比对源报告 sha256 保证文件字节一致（`.sha1`/`.md5`/`.sha256`/`.sha512` sidecar 作为独立 asset 一并搬运），目标仓库名可自定义（默认同源名）；**无需离线 blob store 目录**，补足远程 Nexus（无磁盘访问）的迁移路径。`NexusClient` trait 扩展 `fetch_components` / `download_asset`（流式下载经 `tokio_util::io::StreamReader` 适配），仅 `maven2` hosted 参与、其余整体跳过，单 asset 失败记 WARN 跳过、不中断整批、可重入。迁移不搬运源系统上游凭据（凭据真源 env / 配置，运维另配）。在线拉取的**执行**进一步异步化（FR-83，ADR-0019）：触发端点同步只做枚举源 + 匹配选仓 + 解析凭据后**立即返回 `job_id`（202）**，asset 枚举（先枚举全量得知总数）+ HTTP 下载 + 落地在后台 `tokio` 任务跑、边搬边上报进度；下载 / 写入瞬时失败自动重试、指数退避；进度存 `api` 层进程内有界任务注册表（`MigrationJobs`，经 axum `Extension` 注入各迁移端点；**不落库**，服务器重启即丢失、靠迁移幂等重跑恢复，保留 ADR-0006「无须持久化迁移任务表」），经 `GET /migrate/jobs/{id}` 与 `GET /migrate/jobs` 轮询、支持客户端断点重连。在线拉取任务进一步加**生命周期控制**（FR-91，沿用 ADR-0019 不另写 ADR）：任务与控制端点经注册表共享一份进程内 `JobControl`（std `AtomicBool` 取消 / 暂停标志 + `tokio::sync::Notify` 唤醒，**不引入新依赖**），后台逐 asset 循环在**每个 asset 边界**轮询信号——取消即收尾标 `cancelled`（不算失败、已搬运保留、不再搬后续），暂停即把进度标 `paused` 后在 `Notify` 上**锁外挂起**等待，继续 / 取消唤醒后复核恢复或退出（控制检查 / 等待均在进度锁外，不持锁阻塞 IO）；`OnlinePullPhase` 增 `paused` / `cancelled`、进度增 `paused` 布尔。`api` 层三个薄 handler `POST /migrate/jobs/{id}/{cancel,pause,resume}`（仅 Admin）取注册表中的 `JobControl` 句柄置信号、对已结束任务幂等空操作；控制为协作式（置标志 + 唤醒、后台循环自行退出，不强杀 tokio 任务），只在 asset 边界生效（正在下载的单个 asset 跑完再停）。**离线 blob store 预览异步化**（FR-124，复用 FR-83 注册表）：原同步遍历上万 blob 会在前置反代后超时（504），改为触发端点同步做廉价校验（路径是存在目录）后**立即返回 `job_id`（202）**，`enumerate_blob_store` 在后台 `spawn_blocking` 跑，枚举结果存进度新增字段 `offline_preview`，经同一 `GET /migrate/jobs/{id}` 轮询取回（进程内不落库，与在线拉取共用 `MigrationJobs`）。**离线 proxy/hosted 搬运异步化**（FR-125）：`migrate_nexus_proxy` / `migrate_nexus_hosted` 同步只做在线枚举（失败即 400/502），随后立即返回 `job_id`（202），建仓 + 离线 blob 流式 ingest 在后台任务执行、进度经同一轮询端点观测，并复用 FR-91 取消 / 暂停（`proxy.rs` / `hosted.rs` 抽 `migrate_*_repositories_with_progress` 变体，在每个 blob 与仓库边界经 `online::await_control` 响应控制信号；无进度便捷入口委托之、签名不变）。至此离线迁移全链（预览 + proxy/hosted 搬运）与在线拉取同享异步 job 基建。**统一任务注册表**（FR-131，修订 ADR-0019）：`api::TaskRegistry`（在 `main` 构造、随 `AppState` 共享）把迁移 / 在线更新 / 漏洞库刷新等多类长耗时任务收口为统一轻量记录（`kind` + 统一状态 + 起 / 止 / 更新时间），供 `GET /api/v1/tasks`（活跃 + 近期跨 kind 列表）与 `GET /api/v1/tasks/{id}`（统一记录 + 据 kind 附 `MigrationJobs` / `UpdateJobs` 进度明细）；迁移搬运经其原子单飞门同时只允许一个在途（第二个 `409`、预览不拦），漏洞库刷新经 `vuln::RefreshObserver` 适配器登记（守 `vuln` 不反向依赖 `api`）。进度明细仍归各 kind 专表（单一真源），统一表只存轻量记录、与专表同 `job_id` 关联，进程内不落库、重启即清。
- `protect`：P2 模块——七层（L7）防护中间件链：多维限流（IP/Token/用户/仓库）、并发/连接上限、慢速攻击超时、访问异常检测与自动封禁、IP 黑白名单、CC 挑战、可配置 WAF 规则引擎，并产出防护监控/告警。挂在 `api` 中间件链前段；L3/L4 体积型 DDoS 不在其内。其中**多维限流 + 并发/连接上限（FR-33/51）、慢速攻击超时 + 请求体大小限制（FR-52）、访问异常检测与自动封禁 + IP 黑白名单（FR-53）与 CC 挑战（FR-54）已作为 `api` 中间件落地**（不单设独立 `protect` 模块，见下文中间件机制）；WAF 规则引擎 / 监控告警（FR-55~56）当前形态不实现。
- `protect`：P2 模块——七层（L7）防护中间件链：多维限流（IP/Token/用户/仓库）、并发/连接上限、慢速攻击超时、访问异常检测与自动封禁、IP 黑白名单、CC 挑战、可配置 WAF 规则引擎，并产出防护监控/告警。挂在 `api` 中间件链前段；L3/L4 体积型 DDoS 不在其内。其中**多维限流 + 并发/连接上限（FR-33/51）、慢速攻击超时 + 请求体大小限制（FR-52）、访问异常检测与自动封禁 + IP 黑白名单（FR-53）、可配置 WAF 规则引擎（FR-55）已作为 `api` 中间件落地**（不单设独立 `protect` 模块，见下文中间件机制）；CC 挑战 / 监控告警（FR-54 / FR-56）当前形态不实现。
- `analytics`：P2 能力——使用分析：异步采集访问/下载事件并聚合落 `meta`（SQLite），供数据面板查询；数据本机内部、不外发。采集（FR-57）由 `api::usage` 落地，聚合查询端点（FR-58）由 `api::analytics` 薄 handler 消费 `meta::usage` 的聚合查询入口、面板 UI 由 `web` 控制台展示；不单设独立领域模块。
- `vuln`：P2 模块——漏洞库离线镜像（定期下载 OSV 等公开漏洞数据到本机并落本地库，FR-70）与按制品坐标本地匹配标记（FR-71）均已实现。下载经 `MirrorSource` trait 抽象（生产 `HttpMirrorSource` 走 reqwest 纯 rustls，流式落盘），解压解析每条 OSV 公告后经 `meta` 幂等落库，并支持周期刷新（默认关闭，由配置开启）。坐标级匹配为无副作用纯函数：`format` 各处理器经 `vuln_coordinate` 从制品路径反解生态坐标 `(ecosystem, package, version)`（无标准坐标的 Raw / Docker 不产出、不参与），`meta` 按 `(ecosystem, package)` 查候选受影响行，`vuln::select_hits` 据 OSV `affected` 范围语义（`introduced` 起含、`fixed` 止不含、`last_affected` 止含；另含显式 `versions` 列表）判定命中——全程只比对本机已镜像数据，坐标不外发。Docker 镜像层 OS 扫描更重，留 P3。依赖 `meta` / `config`，方向单向无环。
- `update`：P2 模块——在线自更新编排（管理员手动触发的完整自更新，FR-85 / ADR-0021）。**依赖 `config`，不碰 `meta` / 不读写 DB**，与 `migrate` / `vuln` 同级，由 `api::update` 薄 handler 暴露三个**仅 Admin** 端点 `GET /api/v1/update/check`（查 GitHub 最新稳定 Release、比对版本、返回是否有更新）、`POST /api/v1/update/apply`（按本机 target 下载资产 → 校验 sha256 → 原子替换二进制 → 触发自动重启）与 `POST /api/v1/update/rollback`（FR-104 / ADR-0026：用升级时留的持久回滚备份 `{exe}.rollback.bak` 原子还原上一版 → 触发自动重启；无备份返 409、纯本地操作不受 `enabled` 约束、与 apply 共用单飞互斥）。出站经 `config::build_outbound_client`（FR-84 / ADR-0020 统一出站 helper，honor `[network.proxy]` 的 `http` / `https` scheme 专属代理与 `all` 全 scheme 兜底代理；`all` 经 `reqwest::Proxy::all` 支持 `socks5://` / `socks5h://`，启 reqwest `socks` 特性，注入顺序 `http → https → all`、scheme 专属者优先 all 兜底，FR-100 / ADR-0024），Release 来源经 `ReleaseSource` trait 抽象（生产 `GithubReleaseSource` 走纯 rustls reqwest、测试注入 fake 源不触网）；下载边算 sha256（不整体载入内存）、与同名 `.sha256` 资产定长比对，仅校验通过才原子替换（unix `rename` + 留 `{exe}.bak` / windows 先把运行中 `.exe` 改名 `.old` 再落新文件、下次启动清理）。**出站默认关闭**（`[update] enabled=false` 时端点一律拒绝、绝不联网），仅 sha256 完整性校验不做签名验签，私有仓库可选 token 真源 env `JIANARTIFACT_UPDATE_TOKEN`（不入库 / 不进日志 / 不回显）。仅 FR-86 三发布目标可自更新，其余平台明确报错。方向单向无环。控制台「设置」页（FR-87 只读 + FR-88 可编辑热替换）由 `api::settings` 薄 handler 暴露**仅 Admin** 端点 `GET /api/v1/settings`（读热替换槽当前值）与 `PATCH /api/v1/settings`（编辑网络代理 + 在线更新可调字段，校验后即时生效、无须重启，FR-88 / ADR-0022）。GET 组装脱敏后的网络代理（FR-84）+ 在线更新（FR-85）配置与当前版本——读**运行时可编辑设置热替换槽**（`config::EditableSettings`，非 `state.config`），每代理（http / https / all）拆为 `{ url: 脱敏host, username: 回显用户名, has_password: bool }`——**用户名回显**（连接标识、非密钥）、**密码绝不回显**（仅 `has_password`），更新 token 仅以 `has_token` 暴露（FR-100 / ADR-0024）。PATCH 每代理收 `{ url, username, password(三态：缺省=保留 / 空=清空 / 非空=设置) }`，经纯函数 `rebuild_proxy_url(patch, current)` 据三字段 + 当前存储值重建含凭据存储 URL（改 host 留密码仅当用户名一致时沿用，userinfo 按 RFC 3986 百分号编码），校验后锁外重建出站 client、原子换槽，代理凭据与 token 只入内存槽、不写回 TOML / 不入 DB / 不进日志，重启回落文件 + env 配置（守 ADR-0022 / ADR-0024）。
- `monitor`：P2 模块——主机 / 系统监控采集（FR-98 / ADR-0023）。经 `sysinfo`（裁 features 仅 `system` + `disk`）跨平台采集**这台主机**的基础资源画像：CPU 全局使用率 + 逻辑核数、内存 / 交换的已用 / 总量、磁盘逐盘挂载点 / 总量 / 可用及汇总、系统 uptime。**依赖无（仅用 `sysinfo` 与 `serde`），不碰 `meta` / DB / 网络**，由 `api::monitor` 薄 handler 暴露一个**仅 Admin** 端点 `GET /api/v1/monitor/host`。采样**按请求触发**：单进程共享一份 `sysinfo::System`（refresh 需 `&mut`）经 `AppState` 中 `tokio::sync::Mutex` 串行化，handler 锁内 `refresh` + 读数（纯内存 + 系统调用，无网络 / blob IO，符合「锁外做重 IO」）后组装 DTO 返回；磁盘列表按请求新建刷新。「sysinfo 读数 → DTO」的纯映射（`map_disks` 等）抽为无副作用纯函数便于穷举单测。此**实时快照端点保留**；其历史时序由下条「指标时序」承载（ADR-0027 取代 ADR-0023「不留时序」一条，其余仍有效）。纯本机内部采样、不外发不 phone-home（守 ADR-0009 / 0015 基调）；GET 读取类不入审计（与 FR-97 一致）。CPU 使用率首样可能为 0（需两次采样间隔）属已知取舍。
- **指标时序**（FR-105 / ADR-0027，P2）：统一时序采集与查询，取代 ADR-0023「不留历史时序」。元数据访问层 `meta::metrics` 在 `MetaStore` 上扩展通用扁平时序表 `metric_samples(metric_key, ts, value)`（经 `meta` 唯一入口读写、不绕过 DB），含批量落库、按键 + 范围查询、按保留期 / 行数滚动清理，以及供采样复用的去重存储计数（仓库数 / 去重 blob 数 / 去重字节）。后台采样 / 清理任务 `api::metrics_sampler`（`spawn_metrics_sampler` / `spawn_metrics_retention`）由二进制入口按 `[observability.metrics_timeseries]` 启动：`tokio::time::interval` 按可配间隔每拍采一组 gauge（主机使用率% 复用 `monitor::collect` + `host_percentages` 纯函数、存储 / 仓库计数、防护活跃封禁与限流累计、使用累计访问 / 下载）落库——**锁外做 DB IO**：先锁共享 `System` 取完主机读数即释放锁，再做 `meta` 查询与插入；另一任务按可配保留天数 + 行数兜底周期清理。查询薄 handler `api::metrics_query` 暴露**仅 Admin** 端点 `GET /api/v1/monitor/metrics?metric=&from=&to=&step=`：经 `meta` 取原始样本 → `downsample` 纯函数按步长分桶取平均聚合 → 返回时序点。量级由采样间隔（默认 60s）+ 保留期（默认 7 天）+ 行数上限兜底压住。缓存命中率本期降级不采（proxy 无现成 hit/miss 计数，待埋点）。纯本机内部数据、不外发不 phone-home（守 ADR-0009 / 0015 基调）。
- `licenses`：P2 模块——开源许可清单（FR-102 / ADR-0025）。持有**构建期扫描生成、编译期 `include_str!` 嵌入**的依赖许可清单（Rust crates + 前端 npm，含运行时 + 开发依赖：名 / 版本 / 许可证 / 作者），运行时只读、绝不外发（守 ADR-0009）。**依赖无（仅用 `serde` / `serde_json` / `tracing`），不碰 `meta` / DB / 网络**（定位同 `monitor`），由 `api::licenses` 薄 handler 暴露**公开（匿名可读）**端点 `GET /api/v1/licenses`（不经鉴权门，归因本应公开可查）。清单数据由构建期脚本 `scripts/gen-licenses.mjs` 生成：Rust 侧 `cargo-about`（构建期 CLI，按 `about.toml` 的 accepted 清单扫运行时 crate）+ `cargo metadata` 差集补 dev-only crate，前端侧 `pnpm licenses list`（`--prod` 区分运行时 / 开发），合并写入 `src/licenses/data.generated.json`。「JSON 文本 → DTO」解析与降级抽为无副作用纯函数（`parse`）；惰性缓存交 `embedded()`。仓库内提交 `generated=false` 占位保证恒可编译（仿 `frontend/dist/.gitkeep`），CI 在编译前跑生成步骤产出真实清单；**本地未生成 / 解析失败时降级为空清单 + `generated=false`**（前端显空态），不 panic、不阻断启动。
- `logs`：P2 模块——系统运行日志（FR-107 / ADR-0029）。承载运行时**技术日志**（tracing 的 ERROR/WARN/INFO/DEBUG）的**落盘 + 读取**，**与审计日志（业务留痕落 SQLite，FR-31）严格区分**：本项载体是文件、**不落库**。**依赖无（仅用 `serde` / 标准库 + `tracing-subscriber` 的 `MakeWriter`），不碰 `meta` / DB / 网络**（定位同 `monitor` / `licenses`）。① **文件 sink**：二进制入口 `init_tracing` 保留 stdout 的同时，经 tracing-subscriber 的 `reload` 层（`std` 默认特性，**不引新依赖**）预留一个空文件层槽；`main` 拿到 `data_dir` 后调 `install_file_logging` 把文件层换入，运行日志同时进 stdout 与 `{data_dir}/logs/app.log`（data_dir 确定前的早期日志只进 stdout，属已知取舍）。文件 writer 为 `logs::RollingFileWriter`：单文件追加 + **单次大小滚动**（超阈值 `app.log` → `app.log.1`，旧 `.1` 覆盖），用 std 文件 API 实现、内部 `Mutex` 串行化写与滚动（**不引 `tracing-appender`**）。② **读取端点**：`api::system_logs` 薄 handler 暴露**仅 Admin** 端点 `GET /api/v1/system-logs`，读 `app.log` 全部行 → 调 `logs` 纯函数解析 + 级别精确过滤 + tail（最新在前）+ offset/limit 分页 → 统一分页响应；文件不存在 / 为空返空清单不报错。「日志行 → 结构化条目」解析（`parse_log_line` 取时间戳 / 级别 / 消息，无法识别级别的行不丢、归无级别）、`parse_level`、`tail_filter` 抽为无副作用纯函数便于穷举单测，读文件 IO 留 handler 侧薄封装。纯本机内部数据、绝不外发（守 ADR-0009 / 0015 基调）；GET 读取类不入审计（与 FR-97 一致）。

依赖方向（单向，无环）：

```
api → (auth / authz / repo / format) → (proxy / storage / meta) → config
```

其中 `format` 依赖 `storage` / `meta` / `proxy`；`vuln`（P2）作为后台编排模块依赖 `meta` / `config`，由二进制入口启动其周期刷新任务，落库经 `meta` 唯一入口、不反向依赖上层；`update`（P2）作为自更新编排模块仅依赖 `config`（`api → update → config`），不碰 `meta` / DB；`monitor`（P2）作为主机采集模块仅用 `sysinfo`（`api → monitor`），不碰 `meta` / DB / 网络；指标时序（FR-105 / ADR-0027）采样任务在 `api` 层、经 `meta`（`api → meta`）落库与查询、并复用 `monitor` 读数，守单向依赖、DB 只经 `meta`；`licenses`（P2）作为静态嵌入资源模块仅用 `serde`（`api → licenses`），不碰 `meta` / DB / 网络；`logs`（P2 / FR-107）作为运行日志模块仅用 `serde` / 标准库 + `tracing-subscriber` 的 `MakeWriter`（`api → logs`），不碰 `meta` / DB / 网络（运行日志载体是文件、不落库）。`web` 模块通过 `api` 挂载，向用户提供 React 应用与静态资源。严禁反向依赖与环：上层不被下层反向依赖，`meta` 是元数据的唯一真源。换栈/换框架属于架构决策，须先走新 ADR。

## 3. 数据模型

元数据存于嵌入式 SQLite，是元数据的唯一真源；制品本体（blob）落在文件系统/对象存储，数据库仅存索引与 sha256。

SQLite 核心表（P1，五张）：

- `users`：`id`, `username`, `password_hash`, `role`, `disabled`, `created_at`, `external_idp`, `external_subject`（后两列可空，外部认证身份绑定，仅本地账号为 NULL；仅存非敏感身份标识、不存外部凭据，FR-34 / ADR-0016）
- `tokens`：`id`, `user_id`, `name`, `token_hash`, `created_at`, `last_used_at`, `revoked`
- `repositories`：`id`, `name`, `format`, `type`（`hosted` | `proxy`）, `visibility`（`public` | `private`）, `upstream_url`, `upstream_auth_ref`, `created_at`
- `repo_acl`：`id`, `repo_id`, `user_id`, `permission`（`read` | `write` | `delete` | `admin`，四级动作）
- `artifacts`：`id`, `repo_id`, `path`, `size`, `sha256`, `sha1`, `md5`, `sha512`, `content_type`, `cached`, `created_at`（多摘要并存；blob 寻址仍以 `sha256` 为准，sha1/md5 主要为客户端兼容）
- `audit_log`（P2 审计日志，FR-31 + FR-97 / ADR-0015）：`id`（自增）, `ts`, `actor`, `actor_kind`（`session` | `token` | `basic` | `anonymous`）, `request_id`, `source_ip`, `action`（FR-97 全量非读：所有变更类请求留痕，如 `login` / `token.issue` / `repo.create` / `acl.update` / `group.create` / `settings.update` / `protection.config.update` / `migrate.run` / `update.apply` / `update.rollback` / `artifact.upload` 等，未显式归类的非读路径用兜底 `change.{method}`）, `target_repo`, `target`, `result`（`success` | `denied` | `error`）, `detail`。只记元数据级安全 / 管理事件，不记请求体与制品内容；读取类（GET/HEAD）不入此表；凭据 / 密钥绝不入此表。

使用分析表（P2，FR-57 引入；数据面板 FR-58 经 `GET /api/v1/analytics/usage` 消费）：

- `usage_stats`（访问 / 下载聚合计数）：`repo_name`, `repo_path`（制品仓库内路径，仓库级聚合时为空串）, `action`（`access` | `download`）, `count`, `last_at`；主键 `(repo_name, repo_path, action)`，采集走 UPSERT 累加，并发下计数准确。是长期统计真源。
- `usage_events`（可选明细）：`id`（自增）, `ts`, `repo_name`, `repo_path`, `action`, `actor`（用户名或 `anonymous`，不记凭据）, `source_ip`（可空）。仅在配置 `[observability.usage] detail_enabled = true` 时写入，行数由后台按 `max_detail_rows` 兜底裁剪、删最旧，避免撑爆 SQLite。数据本机内部、默认不外发。

防护告警表（P2，FR-56 / ADR-0017 引入；状态端点经 `GET /api/v1/protection/status` 与 `GET /api/v1/protection/alerts` 消费）：

- `protection_alerts`（防护阈值告警）：`id`（自增）, `ts`, `dimension`（`rate_limit` | `ban` | `cc_challenge` | `waf` | `slowloris`）, `severity`（`warn` | `error`）, `observed_value`（触发时窗内观测计数）, `threshold`（触发阈值）, `window_secs`（评估窗时长）, `detail`（中文文案）。进程内告警评估器在窗内某维度计数达阈值时产一条、异步批量落库；行数由后台按 `[protection.alerts] max_rows` 兜底裁剪、删最旧。**不存任何凭据**，数据本机内部、不外发。
用户组/团队表（P2，FR-49 / ADR-0007 引入）：

- `groups`：`id`, `name`（唯一）, `created_at`（用户组/团队）
- `user_groups`：`group_id`, `user_id`（组成员关系，复合主键；外键指向 `groups.id` / `users.id`，`ON DELETE CASCADE`）
- `repo_group_acl`：`id`, `repo_id`, `group_id`, `permission`（`read` | `write` | `delete` | `admin`，四级动作；对组授予仓库 ACL，结构与 `repo_acl` 对齐但主体为组；外键指向 `repositories.id` / `groups.id`，`ON DELETE CASCADE`）

漏洞库离线镜像表（P2，FR-70 引入；FR-71 坐标级匹配复用同表、不新增制品-漏洞缓存表）：

- `vuln_advisories`：`id`, `source`, `summary`, `details`, `severity`, `modified`, `published`, `created_at`（一条公开漏洞公告一行，来源如 OSV）
- `vuln_advisory_affected`：`id`, `advisory_id`, `ecosystem`, `package`, `ranges`, `versions`（公告受影响坐标逐包展开，`ranges` / `versions` 以原始 JSON 文本保真存储；FR-71 经 `(ecosystem, package)` 索引查候选行、在查询时即时按版本范围匹配，不落制品-漏洞缓存表；外键 `advisory_id` 指向 `vuln_advisories.id`，`ON DELETE CASCADE`）
- `vuln_mirror_state`：`source`, `ecosystem`, `last_refreshed`, `advisory_count`（每来源每生态最近一次成功刷新状态，主键 `(source, ecosystem)`，支持幂等刷新与运维观察）

关系：

- `tokens.user_id`、`repo_acl.user_id`、`user_groups.user_id` 指向 `users.id`。
- `repo_acl.repo_id`、`artifacts.repo_id`、`repo_group_acl.repo_id` 指向 `repositories.id`。
- `user_groups.group_id`、`repo_group_acl.group_id` 指向 `groups.id`。
- `repo_acl` 记录某用户对某仓库的动作授权（`read` / `write` / `delete` / `admin`，高动作蕴含低动作）；`repo_group_acl` 记录某组对某仓库的同类动作授权，组成员经 `user_groups` 继承该授权；`artifacts` 记录某仓库下某路径制品的索引与校验和。

blob 文件系统布局：

- 制品本体保存在数据目录下的 blob 存储区（如 `./data/blobs`），运行期由配置中的数据目录决定其根位置。
- 数据库中的 `artifacts` 行通过 `sha256` 与 `path` 与文件系统中的 blob 关联，数据库本身不保存 blob 二进制内容。
- 制品本体按 `sha256` 内容寻址、前两位分桶（`{sha256[0..2]}/{sha256[2..]}`）。启用可选 S3 后端时，对象 key 沿用同一内容寻址布局（再叠加配置的 `prefix`），SQLite 元数据语义不变、仍为唯一真源（见 ADR-0014）。

敏感项不入 DB 明文：

- 密码以 argon2 哈希形式存于 `users.password_hash`，不存明文。
- API Token 以哈希形式存于 `tokens.token_hash`，不存明文。
- 上游凭据等敏感项不入 DB 明文：数据库仅在 `repositories.upstream_auth_ref` 中存引用，真值走配置/env（如 `config.toml` 或 `JIANARTIFACT_*` 环境变量）。

P2 规划（当前形态不建表、不在数据模型 / 契约中预留占位字段，仅记录演进方向）：

- 七层防护：限流阈值、封禁阈值 / 时长、IP 黑白名单、WAF 规则与告警阈值均走配置（TOML），不与元数据混存。**自动封禁的动态状态（FR-53 已落地）与防护告警的窗内计数 / 去抖状态（FR-56 已落地）保存在进程内内存**（时间窗，重启即清），与登录失败计数同源（见 §3），不落 DB——其为短时自动过期的瞬态信号，无需跨重启持久化，避免为瞬态状态引入新表（ADR-0008 曾提及"封禁状态落 SQLite"作为一种实现取向，落地时按"简单优先 + 与既有登录防护一致"取进程内内存，若后续需跨重启持久化再行评估）。**告警事件本身（达阈值产出的一条记录，FR-56）异步批量落 `protection_alerts` 表**供本地审阅（行数兜底裁剪、不外发，见上表）——与瞬态计数区分：计数 / 去抖在内存、产出的告警落库。
- 使用分析：访问/下载聚合计数（`usage_stats`）与可选明细（`usage_events`，FR-57 已落表，见上）已落地；数据面板展示（FR-58）经 `GET /api/v1/analytics/usage`（仅 Admin）消费聚合计数、控制台「使用分析」页展示，已落地。仅本机内部、不外发。
- 漏洞库：离线镜像的漏洞公告（`vuln_advisories` / `vuln_advisory_affected` / `vuln_mirror_state`，FR-70 已落表，见上）已落地；制品-漏洞匹配（FR-71）在制品详情查询时即时据坐标比对受影响坐标表得出，不落 `artifact_vulns` 等缓存表，坐标级匹配、不外发。

P1 的新增能力不引入新表：制品删除与覆盖/不可变策略作用于既有 `artifacts` 与 blob；登录失败计数保存在进程内存（按账户 / IP），不落 DB；首个管理员写入既有 `users` 表。

## 4. 接口

接口分两层并各自承担一类用途；详细契约见 API.md，此处只给概览与定位。

- 管理 API：挂载于 `/api/v1/*`（涵盖 auth 登录/登出/刷新、当前用户 `me`、users、repositories（含制品浏览与删除）、acl、tokens、search、audit（仅 Admin，P2）、analytics/usage（仅 Admin，P2）、protection/status 与 protection/alerts（仅 Admin，P2）、protection/config（仅 Admin，P2，FR-79：GET 读 / PATCH 改各防护维度配置，即时生效）、health），采用 REST + JSON 风格，统一分页与错误约定（详见 API.md），供 Web 控制台与脚本使用。
- 格式 API：各格式按其原生协议挂载（如 Maven、npm、Docker registry v2、Go GOPROXY、Raw、PyPI Simple Repository API），路径中含仓库名以定位目标仓库；供包管理器客户端按原生协议直接访问。
- Web 控制台：`/` 提供 React 应用，`/assets/*` 提供静态资源；二者均由嵌入二进制的前端产物服务。
- 健康检查：管理 API 下提供健康检查端点。

## 5. 关键机制

- 认证中间件：识别请求携带的 Bearer（先按 JWT 会话校验，失败再按 API Token 哈希校验）、Basic Auth（secret 可为口令 argon2 校验或 API Token 哈希校验）、无 scheme 前缀的裸 Token（Cargo registry 客户端约定，按 API Token 哈希校验）或无凭据，解析出调用方身份（`AuthIdentity`：匿名 / 已认证）注入请求扩展；任何无效凭据回退匿名，禁用用户与已吊销 Token 即时失效。Web 会话以无状态 JWT（HS256）承载，放 `Authorization: Bearer` 头（不走 Cookie，天然规避 CSRF，见 ADR-0011）；JWT 签名密钥真源为数据目录下的 `.jwt_secret` 文件（无则生成 256 位高熵随机密钥，类 Unix 下收紧 0600），绝不入库、不进日志，受 `.gitignore` 的数据目录排除覆盖。API Token 为高熵随机串（`jna_` 前缀），仅签发时返回一次明文，DB 只存其 sha256 哈希，校验走定长比较。
- OIDC 认证集成（P2，FR-34 / ADR-0016）：经统一认证 provider 抽象（`auth::provider::AuthProvider` trait + `AuthenticatedSubject`）接入 IdP；本地口令为默认且始终启用，OIDC 作可选 provider，配置 `[auth.oidc]` 后才实例化（未配置即不存在）。采用标准 **授权码流 + PKCE**：`GET /api/v1/auth/oidc/login` 服务端生成 `state` + PKCE `code_verifier` + `nonce`（按 `state` 暂存于进程内一次性流程表，TTL 10 分钟、取出即消费），重定向到 IdP 授权端点；`GET /api/v1/auth/oidc/callback` 校验 `state`（不存在/过期/已消费即拒，防 CSRF/重放），用 `code` + `client_secret` + `code_verifier` 向 token 端点换 ID Token，**校验签名（JWKS RSA RS256）、`iss`/`aud`/`exp`/`nonce`**（算法锁定 RS256 拒算法混淆），解析外部身份（`sub` + 用户名）。经「外部身份 → 本地用户」映射（见下）得本地用户后，**照常签发既有会话 JWT**（TTL/刷新/登出与 ADR-0011 一致），外部身份只在登录边界出现一次、收敛为本地会话，不在请求热路径回源 IdP；既有四通道与鉴权矩阵零改动。OIDC discovery（`/.well-known/openid-configuration`）与 JWKS 首次使用时拉取并缓存。网络 IO 走纯 rustls 的 reqwest（不引 openssl/native-tls、不引新依赖：JSON 经 `bytes()` + serde_json 解析、ID Token RS256 校验经既有 `jsonwebtoken` 的 `rust_crypto`）。
- LDAP 认证集成（P2，FR-35 / ADR-0016）：经同一认证 provider 抽象接入，作可选 provider，配置 `[auth.ldap]` 后才实例化（未配置即不存在）。采用 **bind 校验**模式：口令型登录（Web 表单 `POST /api/v1/auth/login` 与 Basic Auth，含 Docker v2 令牌端点）在本地口令 / API Token 均未命中时委托 LDAP——用配置 `bind_dn` + bind 口令连接目录，按 `user_search_base` + 过滤模板（`{username}` 占位，按 RFC 4515 转义防注入）搜索唯一用户条目取其 DN，再用该 DN + 用户提交口令做一次 bind；bind 成功即认证通过，外部 `subject` 取用户 DN。经「外部身份 → 本地用户」映射（见下）得本地用户后，**照常签发既有会话 JWT**，外部身份只在登录边界出现、收敛为本地会话，不在请求热路径回源目录；既有四通道与鉴权矩阵零改动，与 OIDC provider 并存不串味。连接走 LDAPS / StartTLS，TLS 由 **rustls（ring）** 提供（`ldap3` 仅启用 `tls-rustls-ring`，不引 openssl / native-tls）；默认不接受明文 `ldap://`（除非运维显式 `allow_insecure`，限可信内网）。空口令前置拒绝（防匿名 bind 误判为成功）。bind 口令真源 env / 配置、绝不入库 / 进日志 / 进 DB 明文；用户提交口令仅用于一次 bind、不留存。
- 外部身份 → 本地用户映射（守 ADR-0010，FR-34 / FR-35 / ADR-0016）：`users` 表新增可空 `external_idp` / `external_subject` 两列，以「provider 类别 + 外部稳定标识（OIDC `sub` / LDAP 用户 DN）」为外部身份键唯一绑定本地用户（仅存非敏感身份标识，**不存任何外部凭据**；本地账号两列为 NULL）。**即时开通（JIT）默认关闭**（`auto_provision=false`）：外部认证成功但本地无对应用户时**拒绝登录**（等价「仅管理员预置账号可登录」）；显式开启后首次外部登录即时建本地用户，**默认角色固定为最低权限 `User`，绝不自动 `Admin`**。外部用户口令哈希为不可校验的占位串，不能经本地口令通道登录。OIDC 与 LDAP 复用同一映射逻辑（`resolve_external_login`），按 provider 类别隔离外部身份键、互不串味。
- 鉴权中间件：按目标仓库 + 操作（读/写）综合判定——综合 public/private 可见性、全局角色、每仓库 ACL 三者；私有仓库对未授权（含匿名）一律拒绝（401/404）。
- 代理缓存与单飞：proxy 仓库缓存未命中时，从上游拉取 → 校验 → 落盘 → 写索引；同一制品的并发请求经单飞合并，避免对上游重复拉取；上游不可用时按策略回退。
- 统一出站网络代理（P2，FR-84 / ADR-0020，运行时热替换见 FR-88 / ADR-0022）：全部出站 `reqwest::Client` 经 `config` 层共享 helper `build_outbound_client(timeout, &NetworkProxyConfig)` 构造，集中注入 `[network.proxy]`（`http` / `https` / `no_proxy`）正向出站代理与既有超时 / rustls / stream 特性。**FR-88 起，构造好的 client 收进随 `AppState` 共享的出站网络热替换槽 `config::NetworkState`（std `RwLock<Arc<NetworkSnapshot>>`，快照含代理配置 + 据其构造的 client）**；各出站点不再持启动期 client，改持 `Arc<NetworkState>`、每次出站经 `network.client()` 取当前 client（读锁内 clone `reqwest::Client` 立即放锁、锁外发请求）。注入点：proxy 回源（`proxy::HttpUpstream`）、Nexus 在线迁移（`migrate::HttpNexusClient`）、漏洞库镜像（`vuln::HttpMirrorSource`，后台周期刷新每次取当前 client）、OIDC（`auth::OidcProvider`）、在线更新（`update::GithubReleaseSource`）。helper 与 `NetworkState` 落在 `config`（所有出站模块共同最底层），不产生反向跨层依赖。设置页 PATCH 改代理后**锁外重建 client、原子换槽**，下个出站请求即用新代理、无须重启。env-vs-config：**配置给值即为真源**（注入即关闭 reqwest 的 auto-sys-proxy，压过系统环境）；**三键全空时不注入**，保持 reqwest 默认行为（含其 honor 系统 `HTTP_PROXY` / `HTTPS_PROXY` / `NO_PROXY`），即「不配置等价现状」。代理 URL 可含凭据，凭据**不入库、不进日志 / 错误信息**（构造失败只记不含凭据的「代理配置无效」类消息），运行时 PATCH 的凭据只入内存槽、不写回 TOML。
- 流式 IO 先落盘再写索引：上传/下载走流式处理，大文件不整体载入内存；写入时先落 blob 并校验 sha256 通过，再写元数据索引，以避免产生孤儿索引。
- 配置 env 覆盖：运行期配置由 `figment` 分层加载——内置默认值 → 单个 TOML 文件 → 环境变量（前缀 `JIANARTIFACT_`，节名后首个下划线映射为嵌套分隔，如 `JIANARTIFACT_SERVER_PORT` → `server.port`），后者优先于 TOML 文件中的同名项。
- 动态配置入库覆盖层（P2，FR-106 / ADR-0028，扩展 ADR-0022）：在上述「文件默认」之上叠加 **DB 覆盖 + 内存槽缓存**，使非密钥配置可面板在线编辑、持久化、重启不丢。覆盖发生在**装配层**（`main` + 新增 `config_overlay` 模块），**`config` 模块本身零 DB 依赖**（依赖方向 `api → meta → config` 不变、无环）：启动序列在 `meta` 就绪后读 `app_settings`（KV 表 `key/value_json/updated_at`，DB 只经 `meta` 的 `load_settings` / `upsert_setting` / `delete_setting`）作 DB 覆盖，经**纯函数** `config_overlay::merge_effective_config(文件默认, env 显式键集合, DB 覆盖)` 合并出生效 `Config` 再填充各热替换槽 / 后台任务参数。优先级**钉死**为 `env 显式 > DB > 文件默认`——某节有 env 显式给值则该节不被 DB 覆盖（部署环境强约束 > 面板改动）。**白名单 + 默认拒绝**守凭据红线：仅 `config_overlay::DYNAMIC_KEYS` 列出的非密钥节可入库 / 被覆盖（`limits` / `protection` / `observability.{audit,usage,metrics,metrics_timeseries}` / `vuln` / `auth` 三个可调标量 / `update` 非密钥字段），`auth` / `update` 经专用非密钥视图序列化，代理账密 / update token / OIDC·LDAP 密钥 / JWT 密钥与 bootstrap 项（`server.*` / `data.*`）结构上不可能入库。`PATCH /api/v1/protection/config` 与 `PATCH /api/v1/settings`（update 非密钥字段）的改动同时 `upsert_setting` 落库、重启仍生效（凭据仍只入内存槽）；合并 / 解析失败只 WARN + 回落文件默认，不阻断启动。
- 七层防护中间件链（P2）：在 `api` 入口前段串接限流（IP/Token/用户/仓库 多维）、并发/连接上限、慢速攻击超时、异常检测与自动封禁、IP 黑白名单、CC 挑战与 WAF 规则匹配；命中即在进入业务前阻断，并产出监控/告警。仅应用层（L7）；L3/L4 体积型攻击交前置设施。
- 运行时防护配置热替换槽（P2，FR-79 / 扩展 ADR-0008）：防护配置的**真源**是随 `AppState` 经 `Arc` 共享的进程内热替换槽 `ProtectionState`（内部 `RwLock<Arc<ProtectionSnapshot>>`，**用 std 实现、不引入外部依赖**）。快照 `ProtectionSnapshot` 收拢「当前生效的 `ProtectionConfig` + 由其编译的派生态（IP 名单匹配器、WAF 规则集）」为一个整体替换的不可变视图。各防护中间件经 `snapshot()` 取快照判定——读锁内仅克隆一个 `Arc` 立即放锁、锁外读配置与派生态（热路径开销可忽略，**锁外做匹配 / IO**）。`PATCH /api/v1/protection/config`（仅 Admin）校验后经 `replace()` **锁外**重建派生态、再短持写锁原子换快照指针，**改完即时生效、无须重启**（这即上文与下列各防护「配置热替换下个请求即按新值判定」的落地机制）；限流计数 / 封禁登记 / 告警去抖等进程内累计运行态为独立构件、**不随配置替换重建 / 清零**。`config.protection` 仅作启动初始装载来源，启动后中间件不再读它（避免双真源）；运行时改动不写回 TOML，重启回落文件配置（配置真源仍是文件 + 环境变量）。详见占位 ADR「运行时防护配置热替换机制」。
- 多维速率限制与并发上限（P2，FR-33 + FR-51 / ADR-0008）：上述七层防护链中**已落地的限流子集**。`api` 路由链中一个单一职责的限流中间件置于身份解析之内（更靠近 handler，需读已注入的身份），按 **IP 维度**、**身份/用户维度（用户 id，含其所有 Token / 会话）** 与 **仓库维度（按格式路径首段仓库名解析，保留前缀 `api`/`v2`/`health`/`metrics`/`assets` 不计入）** 用进程内**固定时间窗计数**判定，任一维度单窗超阈值即在进入业务前返回 `429 Too Many Requests`（带 `Retry-After`）；并按 IP / 用户 / 仓库维度限制**在途并发请求数**，任一维度超并发上限同样返回 `429`。限流计数热路径只取一次 `Mutex` 做整型自增与窗口比较（临界区内无 IO / 无格式化），窗口表过期键在加锁期间按表大小阈值顺带清理、防无界增长；并发计数走**分片 `Mutex`**（按键散列分片降争用），入站 +1、由 **RAII `ConcurrencyGuard`** 在请求结束（含出错 / panic 展开）`Drop` 时 -1，**可靠归还、绝不泄漏在途计数**，计数归零的键随即移除防无界增长。来源 IP 取连接级 `ConnectInfo`（与登录防护一致），**不采信 `X-Forwarded-For` 等可伪造头**——伪造来源 IP 不绕过；轮换 IP 的同一主体仍受身份维度阈值约束。阈值 / 窗口 / 并发上限经 `[protection.rate_limit]` 配置，**默认关闭、阈值保守**（仓库维度与三档并发上限默认 0 = 不启用，不误杀正常包管理器批量并发拉取），配置热替换下个请求即按新值判定。本批为多维限流与并发/连接上限；慢速属 FR-52，CC / WAF / 告警属 FR-54~56，均不在本批。
- 访问异常检测与自动封禁 + IP 黑/白名单（P2，FR-53 / ADR-0008）：上述七层防护链中的封禁与名单子集。`api` 路由链中一个单一职责的中间件置于热路径**前端**（外于身份解析与限流，黑名单 / 封禁检查尽量靠前、先于重处理）。**IP 黑/白名单**经 `[protection.ip_list]` 声明 IP / CIDR（IPv4 / IPv6 均可）：启动时预解析为网段集合（非法项记 WARN 跳过、不阻断启动），**白名单优先级最高**——命中即豁免一切应用层防护、照常进入业务、不参与异常统计；命中黑名单即在进入业务前直接拒 `403`。**访问异常检测 + 自动封禁**：在固定时间窗内按来源 IP 统计**异常信号**（响应为 4xx 客户端错误，含 401/403 鉴权失败与限流产生的 429；5xx 服务端错误不计），单 IP 一窗内异常信号数达阈值即**自动封禁**一个时长，封禁期内该 IP 一律拒 `403`，**窗口到期自动解封**。封禁状态与信号计数为**进程内内存**（与登录失败计数同源，见 §3，重启即清、不落 DB），各经 `Mutex` 保护、并发下计数一致，过期键按表大小阈值顺带清理防无界增长；中间件未启用（名单空 + `ban.enabled=false`）时走零开销快路径直接放行。来源 IP 取连接级 `ConnectInfo`，**不采信 `X-Forwarded-For` 等可伪造头**——伪造来源不绕过黑名单 / 封禁、也不能借伪造头逃避异常统计。阈值 / 窗口 / 封禁时长 / 名单经 `[protection.ban]` 与 `[protection.ip_list]` 配置，**默认关闭、阈值保守宽放**（正常包管理器偶发 404 / 鉴权重试不应触顶），配置热替换下个请求即按新值判定。仅应用层（L7）；CC / WAF / 告警属 FR-54~56，不在本批。
- 慢速攻击超时与通用请求体大小限制（P2，FR-52 / ADR-0008）：上述七层防护链中**已落地的慢速防护子集**。`api` 路由链中一个单一职责的慢速防护中间件置于身份解析**之外**（更靠近连接侧，在读取请求体前介入），把请求体包成带超时与累计字节上限的数据流：等待**首个数据块**超过 `header_timeout`（慢起始）、或相邻数据块间隔超过 `body_read_timeout`（慢 drip）即以 IO 错误**终止流并断开连接**，避免慢速连接长期占用；超时按「**块间空闲**」而非「整体时长」判定——正常持续发数据的大文件流式上传（mvn deploy 大 jar、docker push 大层）不触发，**只惩罚长时间不发数据的 slowloris**。同时对**所有请求**的请求体设可配置**通用大小上限** `max_body_bytes`（区别于仅约束制品上传体的 `limits.max_artifact_size`）：带 `Content-Length` 时在进入业务前即拒 `413`（不读体），分块传输则边读边计、累计超限即以错误断流。中间件未启用时直接放行、零包裹开销；启用时仅给请求体套一层流式计时 / 计数包装（`futures_util::stream::unfold` 逐块惰性处理），不缓冲整个体、不整体载入内存（守流式不变量）；超时 / 超限后立即终止本流、不再 poll 慢速底层流。经 `[protection.slowloris]` 配置，**默认关闭、档位保守**（首块 / 块间超时默认 30 秒、通用体上限默认 0 = 不启用，避免误杀正常大制品流式上传），配置热替换下个请求即按新值判定。仅应用层（L7）：L3/L4 体积型攻击仍交前置反向代理 / CDN / WAF；封禁 / CC / WAF / 告警（FR-53~56）不在本批。
- CC 挑战（工作量证明 PoW）（P2，FR-54 / ADR-0008）：上述七层防护链中**已落地的 CC 缓解子集**。`api` 路由链中一个单一职责的 CC 挑战中间件置于身份解析**之内**（更靠近 handler，须读已注入身份以豁免已认证客户端），对疑似 CC（HTTP 洪水）攻击的**匿名**来源下发**工作量证明（PoW）挑战**：客户端须找到 `nonce` 使 `sha256(challenge_token + ":" + nonce)` 的二进制前导零位数达 `difficulty`，以请求头 `X-CC-Solution: <token>:<nonce>` 带证明重试方放行；无 / 错误证明在进入业务前返回 `429`（错误码 `cc_challenge_required`，响应体携带挑战令牌 / 难度 / 过期 / 提交头名）。**服务端无状态校验**：挑战令牌用 **HMAC-SHA256** 签名（密钥经 `JwtSigner::derive_key` 从会话 JWT 真源密钥**域分隔**派生、**不直接暴露 JWT 密钥本体**、也不与会话 JWT 串味），载荷绑定**连接级来源 IP** + 签发时刻 + 难度，**不存挑战态**（不落 DB、不占内存登记表）；校验经签名（常量时间比对，防时序侧信道）、过期、来源 IP 绑定、PoW 难度四关，一次性弱保证由「短过期 + 绑定 IP」承担（简单优先，不引入有界内存登记）。**触发策略守误杀底线**：**默认关闭**——正常包管理器 CLI（mvn / npm / docker / curl）不会解 PoW，无差别拦截会打断正常匿名拉取；**默认豁免已认证（Bearer / Basic / 会话）请求**，挑战只面向匿名可疑流量。来源 IP 取连接级 `ConnectInfo`、**不采信 `X-Forwarded-For`**（换 IP 的证明不可复用，防绕过）。中间件未启用时直接放行、零开销；启用时匿名请求仅一次 HMAC + 一次 SHA256 校验（无锁 / 无 IO / 无 DB，守热路径低开销）。优先 PoW 而非验证码（CAPTCHA 需第三方 / 前端交互、不适配 API / 包管理器场景且引入外部依赖）。经 `[protection.cc_challenge]` 配置（`enabled` 默认关、`difficulty` 默认 20、`ttl_secs` 默认 300、`exempt_authenticated` 默认 true），配置热替换下个请求即按新值判定。仅应用层（L7）：L3/L4 体积型攻击仍交前置反向代理 / CDN / WAF；WAF / 告警（FR-55~56）不在本批。
- 可配置 WAF 规则引擎（P2，FR-55 / ADR-0008）：上述七层防护链中的**请求模式匹配子集**。`api` 路由链中一个单一职责的 WAF 中间件置于热路径**前端**（与其他 L7 防护同层、在进入业务前）。经 `[protection.waf]` 声明有序规则，每条规则对请求的某个属性字段（**method / path / query / 指定 header**）按三类匹配——**字面（literal，子串包含）/ 通配（wildcard，`*` 任意多字符、`?` 任意单字符，转译为锚定正则）/ 正则（regex，`regex-lite` 子串搜索）**——匹配 `pattern`，命中即执行 `action`（`block` 拒 `403` / `allow` 放行并短路）。规则在**启动期编译一次**（正则预编译、通配转译），热路径只做匹配；**非法规则（字段 / 匹配类型 / 动作非法或正则无法编译）记 WARN 跳过、不阻断启动**，其余规则照常生效。匹配**按声明顺序、首个命中生效**——可用前置 `allow` 给合法模式开豁免口子、再用 `block` 兜底拦截。中间件未启用（`enabled=false`）或空规则集走**零开销快路径**直接放行，不缓冲、无锁、无 IO。WAF 按请求属性匹配，**与来源 IP 无关**——不读 / 不采信 `X-Forwarded-For` 做任何 IP 相关判定（与其他 L7 防护「不采信可伪造头做 IP 项」一致）。规则集随 `AppState` 经 `Arc` 共享，配置热替换重建规则集后下个请求即按新规则判定。**默认空规则集 + 关闭**（不影响现有行为、不误杀正常包管理器请求）。仅应用层（L7）；CC 挑战 / 监控告警（FR-54 / FR-56）不在本批，L3/L4 体积型攻击仍交前置设施。
- 审计日志（P2，FR-31 + FR-97 / ADR-0015）：`api` 路由链中一个单一职责的审计中间件置于身份解析之内（更靠近 handler），运行 handler 后按"方法 + 路径 + 响应状态"归类事件——FR-97 起覆盖从"精选"扩为**全量非读**：**所有变更类请求（POST/PUT/PATCH/DELETE）一律产一条审计**，已知端点归精确语义 `action`（用户 / Token / 仓库 / ACL / 组管理、制品上传 / 删除、设置 / 防护配置热替换、迁移控制 / 搬运、在线更新 apply 等），未显式归类的非读路径走兜底 `change.{method}`（不漏记新增端点），授权拒绝仍记 `denied`（不泄露资源存在性）；**读取类（GET/HEAD，含下载 / 浏览 / 搜索 / 详情）一律不入审计**（交使用分析计数，避免刷屏与性能损耗）；登录事件因需记被尝试用户名，由登录 handler 显式发事件。事件经进程内有界 channel 投递给独立写入任务**批量落 `audit_log` 表**（经 `meta`，不绕过它）；主路径只做一次非阻塞 `enqueue`，**采集 / 写入失败只记 WARN、不影响业务**，channel 满则丢弃 + 计数 + WARN（不反压主路径）。后台轮转任务按 `observability.audit.retention_days` 删旧 + `observability.audit.max_rows` 行数兜底。审计仅 Admin 可经 `GET /api/v1/audit` 分页查询；密码 / Token / JWT / 上游凭据一律不入审计（`actor` 只记用户名，中间件不读请求体与 `Authorization` 值本体，`target` 只取 URL 路径段）。数据本机内部、默认不外发（沿用 ADR-0009 基调）。
- 使用分析异步采集（P2，FR-57 / ADR-0009）：格式 GET 下载（读授权通过的制品 GET）记 `download`、制品详情查看记 `access`；事件经进程内有界 channel 投递给独立写入任务**批量聚合落 `usage_stats`**（经 `meta`，UPSERT 累加，并发下计数准确），主路径只做一次非阻塞 `record`，**采集 / 写入失败只记 WARN、不影响业务**，channel 满则丢弃 + 计数 + WARN（不反压主路径）。明细（`usage_events`）默认关闭，开启后由后台按 `observability.usage.max_detail_rows` 行数兜底裁剪。数据本机内部、**默认不主动外发、不向外部遥测 phone-home**；任何外部导出默认关闭（本批不做导出）。聚合查询入口由数据面板（FR-58）经 `GET /api/v1/analytics/usage`（仅 Admin）消费，返回访问/下载总量、热门制品（按下载）、仓库用量（按下载汇总）等聚合视图；该端点纯查本地聚合、不外发。
- Prometheus 指标端点（P2，FR-32 / ADR-0015）：用 `metrics` facade + `metrics-exporter-prometheus` 进程内 recorder（**pull 模型**，不引外部时序库、不 push / remote-write），启动时若 `observability.metrics.enabled=true` 则安装一次全局 recorder，句柄随 `AppState` 共享。`api` 路由链中一个单一职责的指标中间件置于最内层（最贴近 handler），在请求热路径只做**无锁原子观测**——HTTP 维度（`method` / `status_class` / `format`）计数与延迟直方图、上传 / 下载字节、并发上传 gauge；`proxy` 回源边界（`format` 层）埋点缓存命中 / 未命中、上游回源耗时 / 失败。所有标签取**有界枚举值**，**严禁**以仓库名 / 路径 / 用户名 / 制品坐标作标签（基数纪律，见 ADR-0015）；指标名与标签集中在叶子模块 `metrics_keys`（不依赖业务层，供 `api` 与 `format` 共享，避免魔法字符串散落、防跨层依赖）。渲染仅在 `GET /metrics` 被抓取时发生；端点默认要求认证且仅 Admin，`observability.metrics.allow_anonymous=true` 时免认证抓取（须限内网 / 反代后），`enabled=false` 时返回 404。指标本机内部、不主动外发（沿用 ADR-0009 基调）。
- 使用分析异步采集（P2）：访问/下载事件经异步通道采集与聚合，不阻塞主请求路径，聚合结果落 SQLite 供数据面板查询；数据本机内部、不主动外发。
- 防护监控与告警（P2，FR-56 / ADR-0017）：上述七层防护链的**可观测性收口**，三段构成。① **防护计数接入既有 `/metrics`**：在限流 / 自动封禁 / CC 挑战 / WAF / 慢速攻击的各命中点用 `metrics` 宏原子累加低基数 `jianartifact_` 指标（限流被拒带 `dimension=ip|token|repo|concurrency` 标签、封禁触发 / 当前封禁 IP gauge、CC 下发 / 失败、WAF 阻断、慢速超时；recorder 未装时为 no-op），**严禁以 IP / 仓库名 / 规则模式串作标签**（守 ADR-0015 基数纪律）。② **进程内阈值告警评估器**（`api::alerts::AlertEngine`，随 `AppState` 经 `Arc` 共享）：在固定时间窗内按维度累加防护事件计数，单维度窗内计数达 `[protection.alerts]` 阈值即按严重度记**中文分级日志**（窗内观测值达阈值 5 倍升级 ERROR、否则 WARN）并经有界 channel **异步不阻塞**投递一条告警，由独立写任务批量落 `protection_alerts` 表（经 `meta`，失败仅 WARN 不阻塞、行数兜底裁剪）；同一维度**窗内去抖**（一窗只告警一次、不刷屏），跨窗清零后可再告警；热路径只取一次该维度 `Mutex` 做整型自增与窗口比较（**锁内无 IO、无格式化**，日志与投递在锁外）。计数 / 去抖状态进程内内存（重启即清，见 §3）。③ **管理员只读状态端点**：`GET /api/v1/protection/status`（各维度窗内计数 + 当前封禁 IP 数 + 最近告警）与 `GET /api/v1/protection/alerts`（分页查告警历史），薄 handler 下沉至 `alerts` 评估器 / `meta`，均仅 Admin、**纯本机聚合零外发**。**默认关闭、阈值默认保守宽放**（避免无人值守刷告警与正常高频访问误报），配置热替换下个请求即按新值判定。**告警是本机内部数据：只落本地、经 `/metrics` 被动 pull 或状态端点本地查询，不内置任何外发型通知（Webhook / 邮件等若未来要做须另写 ADR）**。仅应用层（L7）。
- 首个管理员引导：首次启动检测到无任何用户时，从环境变量（`JIANARTIFACT_ADMIN_USERNAME` / `JIANARTIFACT_ADMIN_PASSWORD`）创建首个管理员；未提供则生成随机口令、打印到启动日志（仅首次），要求登录后改密。系统不开放公开自助注册，用户由管理员创建。
- 登录防护：对登录失败按账户 / IP 计数（进程内存，不落 DB），超过阈值在时间窗内临时锁定 / 限流，抵御暴力破解（P2 七层防护提供更强的异常检测与自动封禁）。
- 写入语义与覆盖策略：制品写入按格式应用覆盖 / 不可变规则（如 Maven release 不可覆盖、snapshot 可覆盖）；删除对 hosted 删本体与索引、对 proxy 删缓存。
- Maven 服务端权威元数据（FR-121/122，ADR-0037）：Maven 主版本文件写入 hosted 后，服务端按 `meta` 索引（唯一真源）聚合该 `{groupId}/{artifactId}` 全部版本，写时重生成 artifact 级 `maven-metadata.xml` 并 pom 三级兜底（客户端已传不覆盖 → jar 内嵌提取 → 按 GAV 最小 pom），均经既有 `put_hosted` 物化为布局内真实文件 + 校验和 sidecar；遵循 client-priority（`mvn deploy` 自带 pom 不被改写）。**SNAPSHOT 完整规则（FR-122）**：Web 上传的 SNAPSHOT 主构件由服务端铸造唯一时间戳版本（`{a}-{base}-{yyyyMMdd.HHmmss}-{buildNumber}`，时间戳取真实 now、构建号据目录现状 +1），并据目录内时间戳构建聚合重生成快照级 `{base}-SNAPSHOT/maven-metadata.xml`（snapshot/snapshotVersions/lastUpdated）；`mvn deploy` 自带时间戳构件经同一聚合权威重生成。元数据为索引派生的物化视图、可幂等重写，SQLite 仍是版本真源。
- 列表分页：管理 API 列表端点统一 `offset` / `limit` 分页与过滤，返回 `{ items, total, offset, limit, has_more }` 结构。
- 多校验和：制品写入时同时计算 sha256 / sha1 / md5 / sha512 并入索引；各格式按需提供对应 sidecar（如 Maven 的 `.sha1` / `.md5` / `.sha256`），下载方可据以校验。
- 跨仓库搜索：在 `meta` 的 `artifacts` 索引上做关键字 / 坐标检索，结果按调用方身份过滤——只返回其有读权限的仓库制品，绝不泄露无权私有仓库内容。
- 使用方式片段：`format` 按格式与 `public_base_url` 生成获取与接入片段（Maven 依赖、npm / docker 命令、Raw URL/curl 及仓库接入配置），供详情页展示。
- Web 控制台嵌入与 SPA 回退：`web` 模块经 `rust-embed` 在编译期把 `frontend/dist` 打进二进制；在 `api` 路由链中于 API / 格式 / Docker / 健康检查之后接入——`/assets/{*path}` 提供静态资源（按扩展名推断 Content-Type），其余未匹配 GET 经 `fallback` 回退 `index.html` 交前端客户端路由。前端可深链路由均为单段路径、详情用查询参数，避免与格式 catch-all `/{repo}/{*path}` 冲突。干净检出下 `frontend/dist` 仅含占位、无 `index.html` 时返回 503 提示页，保证后端可独立编译 / 测试（构建顺序：先 `pnpm -C frontend build` 再 `cargo build`）。
- 漏洞库离线镜像（P2，FR-70）：`vuln` 按配置周期把公开漏洞数据集（OSV 等）按生态下载整体镜像（`{base}/{ecosystem}/all.zip`）到本机，流式落盘后解压、逐条解析公告并经 `meta` 幂等落库；下载只携带公开生态名、不把本机制品坐标逐包外发，守隐私红线。默认关闭，由配置 `[vuln]` 显式开启。
- 制品漏洞标记（P2，FR-71）：制品详情查询（`GET /api/v1/repositories/{id}/artifacts/{path}`）时，据仓库格式经 `format::Format::vuln_coordinate` 从制品路径反解生态坐标 `(ecosystem, package, version)`，`meta` 按 `(ecosystem, package)` 查本地候选受影响行，`vuln::select_hits` 用纯函数据 OSV `affected` 范围语义判定命中并去重，详情响应附 `vulnerabilities` 数组（公告 id / 严重度 / 摘要）。无标准坐标的 Raw / Docker 不参与、返回空。全程只比对本机已镜像数据，坐标绝不外发（守 ADR-0012 / 数据不外发）。
- 在线自更新与自重启机制（P2，FR-85 / ADR-0021）：`update` 模块编排「检查 → 应用」两步。**检查**（`GET /api/v1/update/check`，仅 Admin）经 `build_outbound_client`（FR-84，honor `[network.proxy]`）请求 `{api_base_url}/repos/{repo}/releases/latest`，比对当前版本（`CARGO_PKG_VERSION`）与 Release `tag_name`（去前导 `v`，按 `major.minor.patch` 三段整数比较、忽略预发布 / 构建后缀），返回当前 / 最新版本、是否有更新、资产名与发布说明。**应用**（`POST /api/v1/update/apply`，仅 Admin）据 `std::env::consts::{OS, ARCH}` 推导本机 target（仅 FR-86 三目标 `x86_64-unknown-linux-musl` / `x86_64-pc-windows-msvc` / `aarch64-apple-darwin`，其余平台明确报错），按命名契约 `jianartifact-{version}-{target}{ext}` 取资产 → **流式下载边算 sha256**（不二次读盘、不整体载入内存）→ 取同名 `.sha256` 资产定长比对 → 一致才**原子替换**二进制 → 触发**自动重启**。**跨平台原子替换**：Unix 给临时文件置 0755 后 `rename` 同卷原子覆盖（运行中进程持旧 inode 不受影响）、替换前留 `{exe}.bak` 单步回退兜底；Windows 先把运行中 `.exe` 改名 `{exe}.old`（NTFS 允许重命名运行中 exe）再落新文件、失败尽力还原，`.old` 下次启动时清理。替换是最后一步、仅 sha256 校验通过后才进入；**校验失败即删临时文件、保留旧二进制、进程续以旧版运行，永不触碰二进制**。**自动重启经 graceful-shutdown 先于拉起新进程**排掉端口竞争：handler 置进程级 `RestartRequest`（随 `AppState` 共享的 `Arc<RestartHandle>`，内含 `tokio::sync::Notify` + `Mutex<Option<RestartRequest>>`）并触发优雅停机，`shutdown_signal` 经 `select!(ctrl_c, restart.notified())` 两路皆触发停机、drain 在途请求后 `serve` 返回；`main` 据 `restart_mode` 处理——`self`（默认）在端口已释放后拉起新版本：**Unix 用 `exec` 原地替换进程映像**（同 PID / 终端 / 前台作业，tmux 等交互前台自更新后进程不脱离作业控制、日志连续可见，ADR-0032），**Windows 无 `exec` 语义保持 `spawn`+`exit`**（新进程脱离当前控制台）；`exit` 直接退出码 0 交外部进程管理器（systemd / docker）重启（避免与管理器重启叠加成双实例）。**出站默认关闭**（`[update] enabled=false` 时端点一律拒绝、不联网）；只读公开 Release + 下载发布资产，**不外发任何使用数据 / 制品坐标 / 遥测**（守数据不外发红线）；**仅 sha256 完整性校验、不做签名验签**（信任链止于 GitHub Release + HTTPS + sha256）；私有仓库可选 token 真源 env `JIANARTIFACT_UPDATE_TOKEN`，不入库 / 不进日志 / 不回显。下载 / 替换 / 重启的纯逻辑（版本比较、target 推导、资产名推导、替换规划）可单测穷举，**端到端「下载 → 替换运行中二进制 → 自重启 → 新版本起来」依赖真机**（Linux + Windows 各一遍，尤含 Windows `.exe` 运行中改名与重启端口序列；Linux 须在 tmux / 交互前台验 `exec` 原地替换后进程不脱离作业控制、日志连续可见，ADR-0032）。**更新回滚**（P2，FR-104 / ADR-0026，增强 ADR-0021）：apply 在原子替换前把当前运行的二进制持久备份为跨平台一致的 `{exe}.rollback.bak`（单备份、只留上一版，**独立于** Unix `.bak` / Windows `.old`，且**不被启动清理**）；`POST /api/v1/update/rollback`（仅 Admin）校验该备份存在 → 把备份复用同一 `execute_replace` 原子换回当前二进制 → 走既有 graceful-shutdown + 重启链路，无备份返 `409`「无可回滚的备份版本」。回滚是纯本地操作、**不出站**故不受 `[update] enabled` 约束，与 apply **共用单飞互斥**（同一时刻仅允许一个二进制变更在途）；设置聚合视图增 `rollback_available` 供控制台启用 / 禁用回滚按钮。回滚规划（备份源 / 暂存路径推导）为纯函数可单测穷举，端到端「回滚 → 重启 → 版本回上一版」同样依赖真机。**派生临时 / 备份名防 compound**（ADR-0032）：所有 `.new` / `.bak` / `.old` / `.rollback.bak` 派生前先剥离 exe 名末尾已叠的更新管理后缀（正常运行恒等、仅在用户手动跑备份文件的异常路径上收敛），并守自拷贝；启动清理增删「规范名 + ≥2 层管理后缀」的 compound 残留（如 `.bak.bak`），保留单层 `.bak` / `.rollback.bak` 两份。原地替换保留原文件名（更新后文件名仍为旧版串）为**设计行为**——保路径稳定给 service manager / 脚本 / `.rollback.bak` 派生引用，`--version` 自报真实版本。**系统重启 / 关闭**（P2，FR-109 / ADR-0033，扩展 ADR-0021/0032）：`api::system` 薄 handler 暴露**仅 Admin** 端点 `POST /api/v1/system/restart`（按运行时 `restart_mode` 重启、不换二进制）与 `POST /api/v1/system/shutdown`（强制 `RestartMode::Exit` 优雅排空后退出、不自拉起）；两者经现有 `RestartHandle::request_restart` + graceful-shutdown **复用同一停机链路**、不新造，**与 apply / rollback 共用单飞互斥**（同一时刻仅一个进程级变更在途，忙时 409）、操作入审计（`system.restart` / `system.shutdown`）、前端二次确认；纯本地不出站故**不受 `[update] enabled` 约束**（同回滚）。关闭=优雅退出，若部署配了自动重启进程管理器会被其再起（真正停机须经管理器，文档化运维前提）。配套 `PATCH /api/v1/settings` 改为**部分更新**（`network_proxy` / `update` 两块均可选，设置页只发代理、系统页只发更新；向后兼容）。真重启 / 真关闭依赖真机验证。**在线更新异步化**（P2，FR-126，增强 FR-85/87，复用 FR-83 异步 job 基建）：检查 / 应用 / 回滚改为进程内异步 job——触发端点立即返回 `202 + job_id`，下载 / 校验 / 替换在后台执行，进度经 `GET /api/v1/update/jobs/{id}` 按阶段（检查 / 下载 / 校验 / 替换 / 重启）轮询（**不返回假百分比**，根治旧客户端模拟进度卡 95%）；`GET /api/v1/update/check` 改为只读留存的上次检查结果（不联网），`POST /api/v1/update/check` 起后台联网检查 job。更新终态（检查结果 + 上次 apply 结果）留存到 `{data_dir}/update-state.json`、启动早期 `update::backfill_last_apply` 回填，使二进制替换重启后经 `GET /api/v1/update/jobs` 仍可续看——**这是对 ADR-0019「迁移 job 不落库、重启即丢、靠幂等重跑恢复」的有意例外**：更新替换二进制后无法靠重跑恢复，必须留存终态供查看；状态文件**只写非密钥字段，token / 凭据绝不写入**，`update` 模块仍只依赖 `config`、不碰 `meta` / 不读写 DB（守分层不变量）。apply 不支持取消（半截替换会坏二进制），保留 FR-85 单飞互斥与安全门；全过程写 FR-107 系统日志。反代不 504 / 真重启续看 / 真下载替换依赖真机验证。**发布产物压缩包与解压**（P2，FR-138，增强 FR-86/89）：CI 工作流改为每目标同时上传**裸可执行**（保留，过渡期兼容旧版自更新）与**压缩包**（`jianartifact-{version}-{target}.zip`，三平台统一 zip，规避静态 musl ELF 启发式杀软误报），各附 sha256 / md5 校验和文件；zip 内含单个可执行二进制，解压即用。`update` 模块的 `apply_update` 在资产列表中**优先选压缩包**（zip + .sha256 均存在时）：下载 zip → 校验 sha256（校验对象是 zip 整体）→ 解压取出二进制（`zip::ZipArchive`，利用已有 zip 依赖，不引新依赖）→ 原子替换；资产列表中无压缩包时**回落裸 exe 路径**（现有逻辑，向后兼容）。sha256 安全门不变：校验通过才解压替换，校验失败删临时文件、保留旧二进制。

## 6. 部署

- 运行形态：单一可执行二进制 + 一个 `config.toml` 配置文件 + 一个数据目录（存放 SQLite 文件与 blob）。前端已在编译期嵌入二进制，无独立静态资源服务器。首启时若配置文件缺失，二进制自动写一份带中文注释的默认配置到该路径（FR-90，已存在则不覆盖），运维开箱即有可编辑配置。
- 依赖：无需独立数据库或中间件，零外部运行时依赖。
- TLS：可选地将本程序置于反向代理之后，由反向代理做 TLS 终结。
- 跨平台：支持 Linux/Windows/macOS，覆盖 x86_64 与 arm64。

## 7. 关键裁决与不做项

影响架构的重大取舍，详见对应 ADR：

- ADR-0001 技术栈与打包：后端 Rust + axum + tokio，前端 React + Vite + TypeScript（UI 组件库 Mantine）经 rust-embed 嵌入，单一二进制（strip + LTO + `panic = "abort"`、`forbid(unsafe)`）。
- ADR-0002 元数据存储：嵌入式 SQLite 存元数据，blob 存文件系统。
- ADR-0003 认证机制：本地用户名/密码（argon2）+ Bearer Token + Basic Auth + Web 会话/JWT；预留认证 provider 抽象。
- ADR-0004 授权模型：全局角色（Admin/User）+ 每仓库可见性（public/private）+ 每仓库读写 ACL；匿名仅读 public。
- ADR-0005 仓库类型：每格式支持 hosted + proxy（含缓存）。
- ADR-0006 Nexus 迁移：在线 REST API + 离线 blob store 双入口迁移框架，随已实现格式逐期扩展。
- ADR-0019 迁移执行异步化：在线拉取改进程内异步任务（立即返回 job_id + 轮询进度 + 客户端重连，不落库），扩展 ADR-0006。
- ADR-0007 权限粒度与用户组：在授权模型上扩展用户组/团队与细粒度权限动作（read/write/delete/admin），扩展（不取代）ADR-0004。P2。
- ADR-0008 七层防护：应用层防护套件（多维限流、并发/连接控制、慢速攻击防护、异常检测与自动封禁、黑白名单、CC 挑战、WAF 规则引擎）+ 监控告警；L3/L4 交前置设施。P2。
- ADR-0009 内部使用分析：访问/下载统计与数据面板，统计落本地、不外发、不 phone-home。P2。
- ADR-0010 首启管理员引导：首次启动从环境变量或随机口令创建首个管理员，不开放公开自助注册。
- ADR-0011 会话与 JWT 生命周期：会话 / JWT 的 TTL、刷新端点与 CSRF 防护策略。
- ADR-0012 漏洞库离线对接：本地镜像 OSV 等公开漏洞数据 + 坐标级本地匹配，不逐包外发；Docker 镜像层扫描留 P3。P2。
- ADR-0013 Docker Registry v2 Bearer 令牌认证：`/v2/token` 范围令牌端点 + 401 Bearer 质询，复用会话 JWT 的 HS256 密钥；匿名 public 读保持 tokenless，预先 Basic（curl）照旧可用。
- ADR-0014 S3 兼容对象存储后端：经 `BlobStore` 抽象新增可选 opt-in 的 `S3Store`（Cargo `s3` 特性默认关 + 配置 `data.storage.backend`，客户端 aws-sdk-s3 裁 rustls + ring），扩展 ADR-0002，本地文件系统仍默认。P2。
- ADR-0020 统一出站网络代理（**已被 ADR-0022 取代**）：新增 `[network.proxy]`（http/https/no_proxy + env 覆盖）为出站代理唯一真源，`config` 层抽共享出站客户端 helper 统一注入全部出站 reqwest 客户端（rustls 保持、代理凭据脱敏）；配置给值即为真源、不配置保留 reqwest 默认（含系统 env）。其「唯一真源 + 统一 helper + rustls + 脱敏」诸决策仍有效并由 ADR-0022 沿用，仅「运行时不热替换」一条被 ADR-0022 推翻。P2。
- ADR-0021 在线更新（自更新）机制：管理员手动触发，查 GitHub 最新稳定 Release → 按本机 target 下载 → 校验 sha256 → 原子替换二进制（unix rename / windows .old swap）→ graceful-shutdown 排空后自动重启（restart_mode self/exit）；出站默认关闭、仅拉取公开数据不外发、复用 ADR-0020 出站 helper、仅 sha256 完整性校验（不签名）。P2。
- ADR-0022 运行时可编辑设置与出站客户端热替换（取代 ADR-0020）：网络代理 `[network.proxy]` 与在线更新可调字段（enabled/repo/api_base_url/restart_mode/token）经 Admin 在线 PATCH 即时生效、无须重启；`config` 层新增 `NetworkState`（std `RwLock<Arc<NetworkSnapshot>>`，快照含代理配置 + 据其 build_outbound_client 构造的 reqwest::Client），5 处出站点改为持 `Arc<NetworkState>` 按需取 client、PATCH 锁外重建 client 后原子换槽；沿用 ADR-0020 真源 + helper + rustls + 脱敏，凭据只入内存槽不写回 TOML / 不入库 / 不回显。设置页（FR-87）从只读改可编辑（仅 Admin）。P2。
- ADR-0023 主机 / 系统监控采集（扩展 ADR-0015 可观测性）：新增依赖 `sysinfo`（钉版本、`default-features=false`、仅 `system` + `disk`，剔除 component/network/user 控体积）跨平台采集基础主机指标（CPU 使用率 + 核数、内存 / 交换 已用 / 总量、磁盘挂载点 / 总量 / 可用及汇总、uptime）；经**仅 Admin** 的 `GET /api/v1/monitor/host` **按请求单次采样**返回；纯本机内部数据、不外发不 phone-home；CPU 使用率首样可能为 0（需两次采样间隔）属已知取舍。**「不后台轮询、不落库、不留历史时序」一条已被 ADR-0027 取代**（其余决策仍有效；实时快照端点保留）。P2。
- ADR-0024 SOCKS5 出站代理与网页代理凭据管理（取代 ADR-0020「不支持 SOCKS」条目）：启 reqwest `socks` 特性，`[network.proxy]` 新增 `all` 键（env `JIANARTIFACT_NETWORK_PROXY_ALL`）经 `reqwest::Proxy::all` 支持 `socks5://` 全 scheme 兜底代理（注入序 http→https→all）；设置页每代理拆 URL/用户名/密码三字段，**用户名回显、密码三态不回显**，后端纯函数 `rebuild_proxy_url` 据三字段 + 当前存储值重建含凭据 URL（userinfo RFC3986 编码），凭据只入内存槽不落库 / 不回显。P2。
- ADR-0026 自更新回滚（增强 ADR-0021）：升级时把当前二进制持久备份为跨平台一致的 `{exe}.rollback.bak`（单备份、只留上一版，独立于 ADR-0021 临时 `.bak`/`.old`，且不被启动清理）；`POST /api/v1/update/rollback`（仅 Admin）复用 `execute_replace` 原子换回 + 走既有 graceful-shutdown + 重启链路，无备份返 409，回滚纯本地不受 `enabled` 约束、与 apply 共用单飞互斥，设置视图增 `rollback_available`。P2。
- ADR-0027 统一指标时序采集与查询（取代 ADR-0023「不留时序」一条）：通用扁平表 `metric_samples(metric_key, ts, value)` 经 `meta::metrics`（`MetaStore` 扩展、不绕过 DB）落库；后台任务 `api::metrics_sampler` 按 `[observability.metrics_timeseries]`（enabled / sample_interval_secs 默认 60 / retention_days 默认 7 / max_rows 兜底）定时采主机使用率% + 存储仓库计数 + 防护活跃封禁与限流累计 + 使用累计访问 / 下载各域 gauge（锁外做 DB IO），按保留期 + 行数滚动清理；薄 handler `api::metrics_query` 暴露**仅 Admin** `GET /api/v1/monitor/metrics`，`downsample` 纯函数按步长分桶聚合返回时序点；保留 FR-98 实时快照，缓存命中率本期降级不采，纯本机内部数据、不外发不 phone-home。P2。
- ADR-0029 运行时日志文件 sink + 读取 API（扩展 ADR-0015 可观测性）：`init_tracing` 保留 stdout 之外经 `tracing-subscriber` 的 `reload` 层（拿到 `data_dir` 后换入，**不引新依赖 / 新特性**）追加文件 sink 写运行日志到 `{data_dir}/logs/app.log`，`logs::RollingFileWriter` 自实现单文件 + **单次大小滚动**（std 文件 API，不引 `tracing-appender`）；薄 handler `api::system_logs` 暴露**仅 Admin** `GET /api/v1/system-logs`，读文件 → `logs` 纯函数解析（时间 / 级别 / 消息）+ 级别精确过滤 + tail/分页 → 统一分页响应，文件缺失返空不报错；运行日志载体是文件、**不落库**，与审计（业务留痕落 SQLite）严格区分。纯本机内部、不外发。P2。

当前不做项：

- group/virtual 聚合仓库：P3，当前形态不实现，也不在数据模型/契约中预留占位字段。
- OIDC 认证集成：P2，已实现——认证 provider 抽象 + OIDC 授权码流 + PKCE（FR-34 / ADR-0016），外部身份经映射收敛为本地会话，JIT 默认关、默认角色 User。
- LDAP 认证集成：P2，已实现——经同一 provider 抽象的口令型 bind 校验接入（FR-35 / ADR-0016），仅参与 Web 表单 / Basic Auth 口令登录，bind 成功经映射收敛为本地会话，JIT 默认关、默认角色 User；TLS 走 rustls（ring），默认拒明文 ldap://。真机互通（对接 AD / OpenLDAP）待真机验（需 LDAP 目录）。
- S3 兼容对象存储后端：已实现为可选 opt-in 的第二 blob 后端（Cargo `s3` 特性默认关闭 + 配置 `data.storage.backend`，见 ADR-0014）；默认仍为本地文件系统，不启用即零外部运行时依赖。
- 七层防护增强与监控告警：P2，已实现——多维限流 / 并发上限（FR-33 / FR-51）、慢速防护（FR-52）、异常封禁 + IP 名单（FR-53）、CC 挑战（FR-54）、WAF 规则引擎（FR-55）、监控告警（FR-56：防护计数接入 `/metrics` + 进程内阈值告警落 SQLite + 管理员只读状态端点，数据不外发，见 ADR-0017）。L3/L4 体积型 DDoS 始终交前置反向代理 / CDN / WAF。
- 使用分析数据面板：P2，已实现——采集（FR-57）+ `GET /api/v1/analytics/usage` 聚合查询（FR-58，仅 Admin）+ 控制台「使用分析」页展示；统计数据本机内部、不外发，不接任何外部导出。
- 权限增强（用户组/团队、细粒度权限动作）：P2，当前授权仅全局角色 + 每仓库读写 ACL，不预留占位字段。
- 漏洞库对接（离线镜像 + 坐标级匹配）：P2，当前不实现、不预留漏洞相关占位字段；Docker 镜像层 OS 漏洞扫描更重，留 P3。
