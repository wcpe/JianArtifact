# ADR-0009：内部使用分析与数据面板

## 状态

已接受

## 背景

用户需要在控制台看到访问与下载等使用数据（访问量、下载量、热门制品、仓库用量）用于运营与容量判断（即"遥测"）。同时项目设有隐私红线：隐私数据与凭据不得外泄、不得外发到外部平台。需明确"遥测"的口径与数据去向，并把能力定位在 P2。

## 决策

在 P2 提供**内部使用分析**：异步采集访问/下载事件，聚合落本地 SQLite（聚合计数 + 可选明细），由内置数据面板展示。数据为本机内部数据，**默认不主动外发、不向外部遥测平台 phone-home**。如需对接外部可观测平台（如 Prometheus / OTLP），由运维显式开启、默认关闭。

## 理由

- 满足运营对使用数据的可视化需求，同时严守隐私红线。
- 异步采集避免阻塞主请求路径；聚合计数控制存储增长。
- "默认不外发、显式开启导出"在可用性与隐私之间取得平衡。

## 后果

- 正面：开箱即得内部使用洞察，隐私可控。
- 负面/约束：采集须异步且不影响主路径性能，明细数据量级需治理（避免撑大 SQLite）；面板查询要有聚合与时间窗口设计；任何外部导出能力默认关闭，是否开启与去向须在 `OPERATIONS` / `SECURITY` 写清。

## 备选方案

- 主动上报外部遥测平台：与隐私红线冲突。落选。
- 完全不做使用统计：不满足运营诉求。落选。
- 仅用通用指标端点（如 Prometheus）替代：指标偏运维，不等于"访问/下载使用分析"的业务面板；作为可选导出而非替代（默认关闭）。
