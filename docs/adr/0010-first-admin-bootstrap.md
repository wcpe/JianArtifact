# ADR-0010：首个管理员引导（空库自举）

## 状态

已接受

## 背景

系统采用"管理员预置用户"的授权模型——用户由管理员创建，不开放公开自助注册（见 ADR-0004）。但首次部署时数据库为空、无任何用户，需要一条安全的自举路径创建第一个管理员；否则空库无法登录、系统不可用。需在"对自动化/容器部署友好"与"安全（不留出厂弱口令）"之间取舍。

## 决策

首次启动检测到无任何用户时，按以下顺序引导创建首个管理员：

1. 若提供环境变量 `JIANARTIFACT_ADMIN_USERNAME` 与 `JIANARTIFACT_ADMIN_PASSWORD`，据此创建（口令以 argon2 哈希存储）。
2. 否则生成一个随机强口令，创建默认管理员，并把该口令打印到启动日志（仅首次、明文仅此一次），要求首次登录后修改。

系统始终**不开放公开自助注册**，后续用户一律由管理员创建。引导逻辑严格"仅当库中无任何用户"时触发。

## 理由

- 环境变量路径适配容器与自动化部署，无需交互。
- 随机口令兜底保证"零配置也能安全启动"，且**不设固定默认口令**，避免出厂弱口令被扫描爆破。
- 引导只在空库触发，不能被用于后续提权或抢注管理员。

## 后果

- 正面：空库可安全自举、对自动化友好、无固定默认口令。
- 负面/约束：随机口令打印到日志，运维须妥善保管启动日志并尽快改密；env 提供的口令走环境变量而非入库配置（见 `SECURITY.md`）；引导必须严格"仅当无任何用户"触发——对应 `architecture-invariants` 红线"开放公开自助注册或任意人创建管理员"。

## 备选方案

- 固定默认口令（如 admin/admin）：出厂弱口令易被爆破。落选。
- 强制交互式 `--init-admin` 子命令：对容器/自动化部署不友好（需人工一次性介入）。作为可选手段保留，但不作默认。
- 首位注册者自动成为管理员（首启开放注册一次）：存在被抢注风险。落选。
