# ADR-XXXX：运行时防护配置热替换机制

> 占位 ADR（FR-79）：编号 `XXXX` 待主控整合时分配真号（并行开发下不自算 max+1，避免撞号）。
> 本 ADR **扩展 ADR-0008**（七层应用层防护）的「限流阈值与 WAF 规则走配置」一条，补齐其
> **运行时在线热替换**机制，不推翻 ADR-0008 任何决策。

## 状态
提议中

## 背景
ADR-0008 已落地七层（L7）应用层防护，其各维度阈值 / 开关 / 难度 / IP 名单 / WAF 规则来自
`[protection.*]` TOML 配置。原实现把配置以不可变 `Arc<Config>` 在启动期固化进 `AppState`，并在启动期
把 IP 名单、WAF 规则等**派生态**编译为 `Arc<具体类型>`——改任一防护参数都须改 TOML 并**重启进程**。

FR-79 要求 Admin 能在线读取 / 修改各防护维度，**改完即时生效、不重启**（运维遭遇攻击或误杀时需即时
收紧 / 放松，不能依赖重启）。约束（架构不变量）：进程内、无外部 DB / MQ / Redis；`#![forbid(unsafe_code)]`；
**用 std 实现**（不引入 arc-swap 等外部依赖）；锁外做 IO、临界区只护内存态、短持有；防护中间件位于
请求热路径，读路径开销须可忽略。

## 决策
新增一个进程内**运行时防护配置热替换槽** `ProtectionState`，随 `AppState` 经 `Arc` 共享：

- 内部 `RwLock<Arc<ProtectionSnapshot>>`；`ProtectionSnapshot` 收拢「当前生效的 `ProtectionConfig`
  + 由其编译的派生态（IP 名单匹配器、WAF 规则集）」为一个不可变、整体替换的快照。
- 读：`snapshot()` 读锁内克隆一个 `Arc` 立即放锁，调用方在锁外读配置与派生态。
- 写：`replace(cfg)` 先在**锁外**按新配置重建派生态（含 WAF 正则预编译等开销），再短持写锁原子换
  快照指针（写临界区只做一次指针赋值）。
- 防护中间件（限流 / CC 挑战 / 异常封禁 + IP 名单 / 慢速 / WAF）一律改从该槽取快照判定，**不再读
  `config.protection`**——防护配置真源自此为本槽，`config.protection` 仅作启动初始装载来源。
- `PATCH /api/v1/protection/config` 校验后调 `replace` 即时生效；运行态（限流计数 / 封禁登记 / 告警
  去抖 / CC 签名器）为独立构件，**不随配置替换重建 / 清零**。
- 运行时改动**不写回 TOML**：重启回落文件配置，守「配置真源是文件 + 环境变量」。

## 理由
- `RwLock<Arc<快照>>` 是 std 即可表达的「读多写极少」热替换原语：读路径无写争用、仅一次 `Arc` 克隆，
  满足热路径低开销；写路径把编译开销放在锁外，临界区极短。无须引入 arc-swap 等外部依赖（守简单优先 +
  零外部依赖）。
- 把「配置 + 派生态」打包成**整体替换的快照**，保证两者始终一致——绝不出现「配置已换、派生态仍是旧的」
  中间态（穷举并发 replace + snapshot 的自洽性测试覆盖）。
- 只让 `protection` 子树可热替换、不动 `config: Arc<Config>` 字段：非防护配置（server / data / auth /
  limits / proxy / observability / vuln）读取点零改动，改动面收敛在防护模块内，不越界。
- 运行态与配置态分离：改一次配置不应放空已积累的防护状态（限流计数 / 封禁记录 / 告警去抖），故运行态
  不随配置重建。

## 后果
- 正面：防护参数在线可调、即时生效、无须重启；配置与派生态强一致；零外部依赖、热路径开销可忽略。
- 约束 / 负面：`AppState` 多一个 `protection` 槽；防护配置真源从「`Arc<Config>` 不可变快照」转为「热替换
  槽」，`config.protection` 退化为仅初始装载来源（须避免再有读取点读它，防双真源漂移）；运行时改动不持久化，
  重启回落文件配置（运维须知；写入 CONFIG / OPERATIONS）。

## 备选方案
- **改 `config` 字段为 `Arc<RwLock<Arc<Config>>>`**：会波及全代码库几十处 `state.config.xxx` 直接字段
  访问（含大量非防护 handler），改动面巨大且越界。落选。
- **引入 `arc-swap`**：能更优雅地无锁热替换，但属新增外部依赖，违背「用 std 实现 / 零外部依赖」约束。落选。
- **改完 TOML 触发 SIGHUP 重载整个 `Config`**：仍是「重启式」重载、且重载面过宽（牵动所有子系统派生态），
  不满足「在线 PATCH 单维度即时生效」且风险大。落选。
