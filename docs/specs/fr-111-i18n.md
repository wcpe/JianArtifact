# 功能规格：全站国际化（i18n）迁移

> 状态：开发中　·　关联 PRD：FR-111　·　ADR：ADR-0034

## 1. 背景与目标

P2。前端文案全硬编码中文、审计动作 key 显原始串。引入 i18next + react-i18next，把全站文案抽到中文 locale、动作 key 译中文，并为多语言扩展预留结构（本期只交付 zh-CN）。这是基建先行项：后续 FR-115~119 在其上写 i18n-native 代码。

## 2. 需求（要什么）

- **范围内**：i18next 框架落地（init + provider + 测试 setup）；全站页面 / 组件硬编码业务文案迁入 `locales/zh-CN/<ns>.ts`；审计 / 近期活动动作 key → 中文标签（`tAuditAction`）；key 命名规范稳定可扩展。
- **不做（范围外）**：多语言文案（仅 zh-CN）；语言切换 UI；后端 / API 改动；把日志 / 注释 / 测试夹具串也 i18n（仅用户可见 UI 文案）。

## 3. 设计（怎么做）

见 ADR-0034。结构：`src/i18n/index.ts`（init + `tAuditAction`）+ `locales/zh-CN/<ns>.ts`（每页一个 ns + common/nav/errors/auditActions）。组件 `useTranslation('<ns>')` → `t('key')`，跨 ns 用 `t('common:x')`。**文案逐字不变**（locale value=原串），测试基本不动。

## 4. 任务拆分

- [x] 引入 i18next/react-i18next；建 i18n/index.ts + ns 文件骨架；wire main.tsx + 测试 setup
- [x] ADR-0034 + 本 spec
- [x] 试点：近期活动卡（动作 key→中文）+ 测试转绿
- [ ] 分批迁移其余页面 / 组件文案入各 ns（并行子代理，各只动自己 ns + 页面）
- [ ] 校验：无残留硬编码业务文案、全量 vitest 绿、build 绿
- [ ] 文档同步：PRD 状态、ARCHITECTURE（前端 i18n 机制）、CHANGELOG

## 5. 验收标准

- 全站用户可见 UI 文案经 `t()` 渲染、集中在 `locales/zh-CN/`；无残留硬编码业务中文（注释 / 日志 / 测试夹具除外）。
- 审计 / 近期活动动作显示中文标签（artifact.upload→上传制品 等），未知动作回落原串。
- 既有 vitest **全量绿**（文案逐字不变、断言可视中文者不破；断言原始 key 者改断言译后标签）。
- `pnpm -C frontend build` 绿；`pnpm lint` 净。
- 不改后端 / API 契约。

## 6. 风险 / 待定

- 并行迁移防冲突：每页只动自己 ns 文件 + 页面，**不动** index.ts/common.ts（共享文件由主控统一管），新共享串先放各自 ns。
- 真机视觉：迁移后界面文案应与迁移前逐字一致（无视觉变化），发版前真机扫一眼。
