# 功能规格：仓库详情页文件浏览重构 + 两树修正

> 状态：开发中　·　关联 PRD：FR-115（增强 FR-76/67）　·　分支：feature/fr-115-browse-redesign

## 1. 背景与目标
用户反馈仓库详情页「浏览」标签的文件树体验不佳：左树太窄、整页随内容滚动、文件树没有明显的层级递进、长文件名（同目录下 `.jar` / `.jar.md5` / `.jar.sha1` / `.jar.sha256` / `.jar.sha512`）被截断难以辨认。属 P2 UX 增强，纯前端。

## 2. 需求（要什么）
- 仓库详情页浏览 tab 改为「左侧文件树为主、右侧详情为辅」：左树占主宽、右详情为辅。
- 页面固定高度（不随内容整页滚动），左树与右详情各自独立滚动；树铺满内容区可用高度。
- 文件树逐层缩进，使各层级在视觉上明显递进（每深一层缩进更多）。
- 文件名不截断：去掉导致截断的 `truncate` / 固定窄宽，全名可辨（换行或横向滚动）。
- 搜索页结果树同步修「层级递进缩进 + 不截断」。
- 范围内：上述两处文件树的布局与渲染样式调整、对应 i18n 文案（如有新增）。
- 不做（范围外）：sidecar 折叠、单子目录链折叠（YAGNI，用户未要）；不改后端 / API 契约。

## 3. 设计（怎么做）
- 纯前端，无架构决策，不写 ADR。
- `RepositoryDetailPage.tsx` 的 `BrowseTab`：
  - 外层容器设固定高度（`calc(100vh - 顶部偏移)`），`display:flex` 两栏。
  - 左树为主栏（`flex` 偏大 / 更宽），右详情为辅栏；两栏各自 `overflow:auto` 独立滚动。
  - 左树 `ScrollArea` 铺满容器高度（`h="100%"`），不再用固定 `mah={520}` 与固定 `w={320}`。
- 文件树缩进：抽出每层缩进步长常量，加大步长使递进明显；缩进施加于行整体（含缩进引导线感）。
- 文件名：去掉 `Text truncate`，改为允许折行（`Text` 无 `truncate`、不固定窄宽）。
- `SearchPage.tsx` 的 `TreeNodes`：同样去 `truncate`、用同一缩进步长加大递进。
