# 功能规格：角色感知 UI（匿名访客 + 普通用户页面）

> 状态：开发中　·　关联 PRD：FR-95　·　分支：feature/fr-95-role-views

## 1. 背景与目标

当前 Web 控制台偏管理员视角，且 `App.tsx` 的 `RequireAuth` 把**所有**路由都挡在登录之后（未登录一律跳 `/login`）。但后端早已支持**匿名读 public 仓库**（FR-08，已交付）与**普通用户按 ACL 访问**（FR-13/ADR-0004）。本 FR 属 Web UX 重构 epic（B2b 收尾，P2），把这套既有权限模型在**前端**也呈现出来：匿名访客无需登录即可浏览公开制品（只读），普通用户登录后按 ACL 看可见仓库 / 上传 / 自助管理自己的 Token，管理员维持现状全套。纯前端，不改后端（权限后端已强制）。

## 2. 需求（要什么）

- **三层路由守卫**：把 `App.tsx` 路由分为三层——**公开（匿名可达，只读公开内容）/ 需登录（user+）/ 需管理员（admin）**。匿名访问公开路由不再被强制跳 `/login`；匿名访问需登录 / 管理路由仍跳登录（带回跳）；普通用户访问管理路由仍重定向走。
- **匿名访客视图**：未登录即可看一个公开浏览视图——列公开仓库、浏览 / 搜索 public 制品（只读）；页眉显眼「登录」按钮（而非用户菜单）；**不显示** 上传 / 管理 / 设置等入口；私有内容不出现（后端已过滤，前端不强求）。
- **普通用户视图**：登录后按 ACL 看可见仓库（读 / 写）、上传、自助管理自己的 Token；**不显示** 全局管理入口（用户 / 组 / 防护 / 监控 / 迁移 / 设置）；管理路由对其不可达（跳走），非管理路由可达。
- **管理员视图**：维持现状（全套导航与页面）。
- 范围内：`App.tsx` 三层守卫；`AppLayout` 匿名态页眉（登录按钮 + 公开导航项），登录态沿用 FR-92 折叠导航 + `isAdmin` 门控。
- 不做（范围外）：不改后端任何端点与权限判定；不做注册 / 公开自助开户（PRD 明确不开放）；不碰设置 / 监控 / 迁移等管理页内部逻辑；不破坏 FR-92（折叠导航 / 角色门控 / 段精确高亮 / 密度）/ FR-93（浏览树）/ FR-94（页眉搜索）/ FR-99（监控整合）；不新增前端依赖。

## 3. 设计（怎么做）

- **三层守卫**（`App.tsx`）：
  - 公开层：`/repositories`、`/repository`、`/search`、`/artifact`、`/`（落地）——匿名与登录用户皆可达。匿名走 `AppLayout` 的匿名 shell，只读浏览公开内容；后端按身份过滤返回，前端不额外加私有判定。
  - 需登录层：`/tokens`、`/upload`——`RequireAuth` 守卫，匿名跳 `/login`（带 `from` 回跳）。
  - 需管理员层：`/users`、`/groups`、`/protection`、`/monitor`、`/migration`、`/settings`——`RequireAuth` + `RequireAdmin`；匿名跳登录、普通用户重定向到 `/`。
  - `RequireAuth` 在恢复会话 `loading` 期间显示加载态（沿用现状）；匿名（`loading=false && user=null`）下公开路由不再被它拦截——故公开路由不包 `RequireAuth`，由 `AppLayout` 自身据 `user` 渲染匿名 / 登录 shell。
  - 落地路由 `/`：匿名 → 重定向到 `/repositories`（公开浏览落地）；登录 → 仍是 `DashboardPage`（仪表盘读 `user`，不能匿名渲染）。
- **`AppLayout` 角色感知 shell**：`AppLayout` 已用 `useAuth`，扩展为据 `user` 是否为 null 分两态：
  - 匿名态：页眉右侧显「登录」按钮（`navigate('/login')`），不显示用户名 / 登出；导航只留公开项（仓库管理 / 制品搜索）；页眉搜索保留（FR-94，匿名调既有 `/search` 拿 public 结果）。
  - 登录态：沿用现状——用户名 + 角色 + 登出，导航按 `isAdmin` 门控（FR-92），不回归。
  - 导航项清单加 `publicVisible` 标记区分「匿名可见的公开项」；过滤逻辑：匿名只留 `publicVisible`，登录用户按现有 `adminOnly` 规则。
- 复用既有页面：公开浏览 / 搜索复用 FR-93/94 的 `RepositoriesPage` / `RepositoryDetailPage` / `SearchPage` / `ArtifactDetailPage`，它们已据 `isAdmin` 门控写操作按钮（匿名 / 普通用户 `isAdmin=false`，天然只读 / 无管理按钮），无需改页面内部逻辑。

## 5. 验收标准

- 匿名访问公开路由（`/repositories` / `/search` / `/repository` / `/artifact`）不跳登录、能渲染公开浏览 / 搜索；页眉有「登录」按钮；看不到任何管理 / 上传 / 登出入口（断言导航与页眉）。
- 匿名访问需登录路由（`/tokens` / `/upload`）与管理路由 → 重定向到 `/login`（断言落到登录页）。
- 普通用户：能达公开路由与 `/tokens` / `/upload`；访问管理路由（如 `/users` / `/settings`）→ 重定向到 `/`（断言不可达）；导航无管理入口（沿用 FR-92 既有断言）。
- 管理员：全套导航与全部路由可达（沿用既有断言，不回归）。
- FR-92/94/99 既有用例全绿不回归（折叠导航、角色门控、页眉搜索跳转、段精确高亮、监控整合）。
- 未新增前端依赖；未改后端、未做自助注册。

## 6. 风险 / 待定

- 匿名落地选 `/repositories`（复用既有公开仓库列表页）而非新建落地页——保持简单、不新增页面；`RepositoriesPage` 空文案在无公开仓库时即为友好提示，足够。
- 前端不对私有内容做二次判定（后端已强制过滤），避免与后端真源重复、防止前后端判定漂移。
