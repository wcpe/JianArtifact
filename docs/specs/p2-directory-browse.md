# 功能规格：目录列表 API + 仓库索引视图 + Web 文件浏览器

> 状态：开发中　·　关联 PRD：FR-75 / FR-76　·　分支：feature/fr-75-browse

## 1. 背景与目标

仓库内现有制品只能逐条按已知路径下载（`GET /{repo}/{path}`），无法浏览目录结构。本规格补齐两条能力（P2）：

- **FR-75**：让仓库路径支持目录浏览。以 `Accept` 头驱动双形态——浏览器（`Accept: text/html`）拿到类 Apache 目录索引的 HTML 页；API 调用（`Accept: application/json`）拿到 JSON 目录项列表。
- **FR-76**：Web 控制台在仓库详情页内嵌「文件浏览」标签，可逐级展开目录、点文件看详情 / 下载。

## 2. 需求（要什么）

### 范围内
- 后端目录列举端点：复用既有 catch-all 路由 `GET /{repo}/{*path}`，以**路径尾斜杠**作为「目录请求」信号（`GET /{repo}/dir/`、仓库根 `GET /{repo}/`）；无尾斜杠仍为单文件下载（行为不变）。
- 双形态：目录请求按 `Accept` 协商——含 `application/json` 返回 JSON 目录项；含 `text/html` 返回 HTML 索引页；两者皆无时默认 JSON。
- 目录项语义：在给定前缀下**只下钻一层**，把同前缀的制品折叠为「直接子目录 + 直接文件」两类条目（类文件浏览器），不做整棵子树的扁平铺开。
- 鉴权：private 仓库要求有效身份（Basic / Bearer / 会话任一）且有读权限；**匿名 / 无权访问 private 一律拒绝**，返回 404 且不泄露资源存在性；结果按读权限过滤（仅返回调用方可读仓库内的目录项）。
- 前端：仓库详情页新增「文件浏览」tab，按目录树逐级浏览；文件条目可跳详情 / 下载。

### 不做（范围外）
- 不做跨仓库的全局目录树（跨仓库检索是 FR-67，已交付）。
- 不做目录级的写 / 删 / 重命名操作（删除是 FR-60，按单制品路径）。
- 不做分页式的超大目录虚拟滚动（一次列举当前目录层，足够 MVP/P2）。
- 不引入新存储 / 新查询引擎；前缀列举走既有 SQLite。

## 3. 设计（怎么做）

属 ADR-0005（仓库类型）范畴内的浏览能力，**无需新 ADR**。

- **meta 层**：新增按 `repo_id` + 路径前缀列举制品的查询（`SELECT ... WHERE repo_id = ? AND path LIKE ?||'%' ESCAPE ...`，转义 `%`/`_`，按 path 升序）。meta 仍是元数据唯一入口。
- **format 层**：新增纯函数把「前缀 + 命中制品列表」折叠为一层目录项（区分子目录 / 文件，去重子目录名、给文件带 size/sha256/时间）。纯函数便于穷举测试。
- **api 层（薄 handler）**：在 `get_artifact` 入口按尾斜杠判定目录请求，分出列举分支：
  - 读授权复用 `load_readable_repo_by_name`（private 对无权 → 404 隐藏存在性）。
  - 列举后按 `Accept` 渲染 JSON 或 HTML（HTML 仅做模板拼接，转义用户可见文本防注入）。
  - 单文件分支保持原样。
- **前端**：`api/{types,endpoints}.ts` 加目录列表类型与端点；`RepositoryDetailPage.tsx` 加「文件浏览」tab，组件逐级请求目录、渲染 Mantine 列表；文件点击走既有制品详情 / 下载。

## 4. 任务拆分
- [ ] meta：按前缀列举制品查询 + 单元测试（前缀转义、升序）
- [ ] format：目录项折叠纯函数 + 单元测试（子目录去重、一层下钻、根目录）
- [ ] api：`get_artifact` 目录分支 + Accept 协商 + HTML 渲染；薄 handler
- [ ] 后端集成测试：JSON / HTML 双形态；**鉴权过滤**（匿名 / 无权列举 private → 404 不泄露存在性）；public 匿名可浏览；结果按读权限过滤
- [ ] 前端：类型 / 端点 / 「文件浏览」tab 组件 + 测试
- [ ] 文档同步：PRD 状态、API.md 目录端点（双形态）、CHANGELOG 未发布段、必要时 ARCHITECTURE

## 5. 验收标准
- 后端 `cargo test` 全绿，且含以下穷举：
  - 目录请求 `Accept: application/json` → 200 + JSON 目录项数组（含子目录与文件，folder/file 类型与文件元数据正确）。
  - 目录请求 `Accept: text/html` → 200 + `text/html` 索引页（含条目链接）。
  - **鉴权**：匿名列举 private 仓库目录 → 404（JSON 与 HTML 两形态都不泄露存在性）；无读 ACL 登录用户列举 private → 404；有读 ACL → 200。
  - public 仓库匿名可列举目录。
  - 无尾斜杠路径仍走单文件下载（回归不破）。
  - 前缀列举只返回该前缀下的项，不串入兄弟前缀（`docs/` 不匹配 `docsx/`）。
- 前端 `cd frontend && npm test` 通过、`npm run build` 成功。
- **实机维度**：浏览器手动打开 `GET /{repo}/dir/` 看 HTML 索引、Web 控制台「文件浏览」tab 实操——worktree 内不长跑服务，**标「待真机验」**，以 axum 内嵌集成测试与前端构建 / 单测覆盖协议与渲染。

## 6. 风险 / 待定
- 尾斜杠作为目录信号需与各格式原生协议路径不冲突：本端点仅服务通用浏览，原生协议（npm/docker/...）各自的尾斜杠语义由其分派分支先行处理，目录分支只在通用下载回退前介入——需保证不吞掉原生协议路径。设计上目录分支放在格式原生分派**之后、通用单文件下载之前**，仅当通用格式（如 raw/maven）且路径以 `/` 结尾时触发。
- HTML 渲染须转义文件名 / 路径，防止存储型 XSS。
